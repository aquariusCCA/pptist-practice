# 簡報與頁面資料

這一篇要解決的核心問題是：

- `slides store` 裡到底保存了什麼？
- `Slide` 和 `SlidesState` 有什麼差別？
- 一頁投影片除了元素之外，還有哪些頁面層級資料？

對應原始碼：

- `PPTist-SourceCode/src/types/slides.ts`
- `PPTist-SourceCode/src/store/slides.ts`

---

## 先一句話理解

- `SlidesState`：**整份簡報在編輯器中的主要狀態容器**
- `Slide`：**單頁投影片的內容本體**

也就是說：

- `SlidesState` 管的是整份簡報與編輯器視角
- `Slide` 管的是單頁真正要展示與保存的內容

這兩個型別雖然都和投影片有關，但層級不同。

---

## `SlidesState`

`SlidesState` 是 `slides store` 裡的主要 state。

它通常包含：

- `title`
- `theme`
- `slides`
- `slideIndex`
- `viewportSize`
- `viewportRatio`
- `templates`

可以先把它理解成：

> 編輯器目前手上這份簡報的總狀態。

---

## `SlidesState` 各欄位的角色

### `title`
- 簡報標題
- 屬於整份簡報層級，不是單頁層級

### `theme`
- 簡報主題樣式
- 型別是 `SlideTheme`
- 會影響整份簡報的預設視覺設定
- 實際欄位包含：
  - `backgroundColor`
  - `themeColors`
  - `fontColor`
  - `fontName`
  - `outline`
  - `shadow`

### `slides`
- 所有投影片資料
- 型別可理解為 `Slide[]`
- 這是整份簡報最核心的內容區塊

### `slideIndex`
- 目前正在編輯或聚焦的頁面索引
- 這是編輯器狀態，不是投影片內容本身

### `viewportSize`
- 視窗基準寬度
- 主要用於畫布縮放與顯示計算

### `viewportRatio`
- 視窗比例
- 預設值為 `0.5625`
- 這在畫布計算裡代表寬高比 `16:9`

### `templates`
- 範本清單
- 提供建立新頁面或套用內容時的模板來源

---

## 原始碼中的預設值

在 `src/store/slides.ts` 中，常見預設值大致是：

- `title = 未命名演示文稿`
- `slides = []`
- `slideIndex = 0`
- `viewportSize = 1000`
- `viewportRatio = 0.5625`
- `templates = 4 個內建範本`

`theme` 也有預設結構，通常包含：

- `themeColors`
- `fontColor`
- `fontName`
- `backgroundColor`
- `shadow`
- `outline`

這裡要注意一個觀念：

> `theme` 雖然也屬於簡報資料，但它更偏向整份簡報的全域樣式設定，而不是單頁內容本身。

---

## `SlidesState` 和 `Slide` 的關係

可以先用下面這個簡圖理解：

```text
SlidesState
├─ title
├─ theme
├─ slides: Slide[]
├─ slideIndex
├─ viewportSize
├─ viewportRatio
└─ templates
```

其中最重要的是：

```text
slides: Slide[]
```

也就是說，`SlidesState` 會持有多個 `Slide`。

---

## `Slide`

`Slide` 是單頁投影片的資料結構。

它代表的是：

> 一頁投影片真正被保存下來的內容。

常見欄位有：

- `id`
- `elements`
- `notes?`
- `remark?`
- `background?`
- `animations?`
- `turningMode?`
- `sectionTag?`
- `type?`

---

## `Slide` 各欄位的角色

### `id`
- 投影片 ID
- 用來識別這一頁

### `elements`
- 這一頁上的所有元素
- 型別可理解為 `PPTElement[]`
- 這是頁面內容最核心的欄位

### `notes?`
- 投影片註解
- 型別是 `Note[]`
- 用來保存註解資料，而不是頁面主內容

### `remark?`
- 頁面備註文字
- 型別是 `string`
- 這是一段純文字備註，不是結構化註解資料

### `background?`
- 投影片背景
- 型別是 `SlideBackground`
- 實際欄位包含：
  - `type`
  - `color?`
  - `image?`
  - `gradient?`

### `animations?`
- 元素動畫設定
- 型別是 `PPTAnimation[]`
- 每筆動畫的欄位包含：
  - `id`
  - `elId`
  - `effect`
  - `type`
  - `duration`
  - `trigger`

### `turningMode?`
- 投影片轉場模式
- 型別是 `TurningMode`
- 屬於頁面切換時的表現設定
- union 值包含：
  - `no`
  - `fade`
  - `slideX`
  - `slideY`
  - `random`
  - `slideX3D`
  - `slideY3D`
  - `rotate`
  - `scaleY`
  - `scaleX`
  - `scale`
  - `scaleReverse`

### `sectionTag?`
- 分段標記
- 型別是 `SectionTag`
- 用來表示這頁所屬的段落或章節資訊
- 實際欄位只有：
  - `id`
  - `title?`

### `type?`
- 投影片類型
- 型別是 `SlideType`
- 用來標示這頁在整份簡報中的角色，例如封面、目錄、內容頁、結尾頁等
- union 值包含：
  - `cover`
  - `contents`
  - `transition`
  - `content`
  - `end`

---

## 頁面資料的最小關係圖

```text
Slide
├─ id
├─ elements: PPTElement[]
├─ notes?: Note[]
├─ remark?
├─ background?: SlideBackground
├─ animations?: PPTAnimation[]
├─ turningMode?
├─ sectionTag?: SectionTag
└─ type?
```

如果只記一句話，就是：

> `Slide` 不是只有 `elements`，它還會帶著背景、動畫、註解、分段與頁面類型等頁面層級資訊。

---

## 相關巢狀型別

### `Note`

`notes` 不是單純的字串陣列，而是結構化註解資料。

常見欄位有：

- `id`
- `content`
- `time`
- `user`
- `elId?`
- `replies?`

`NoteReply` 也是一個獨立結構，欄位包含：

- `id`
- `content`
- `time`
- `user`

其中：

- `elId?` 表示註解可能對應到某個特定元素
- `replies?` 的型別是 `NoteReply[]`

這代表 `Note` 不只是備註文字，而比較像可被追蹤、可被回覆的註解資料。

---

### `SectionTag`

`sectionTag` 不是單純字串，而是具有結構的資料。

常見欄位：

- `id`
- `title?`

所以它表示的不是只有一個標籤名稱，而是一個可以被識別的分段資訊。

---

### `SlideBackground`

背景資料是結構化物件，不只是單純的顏色字串。

`SlideBackground` 的欄位：

- `type`
- `color?`
- `image?`
- `gradient?`

其中 `type` 的 union 值是：

- `solid`
- `image`
- `gradient`

`image?` 的型別是 `SlideBackgroundImage`，其欄位是：

- `src`
- `size`

`size` 的 union 值是：

- `cover`
- `contain`
- `repeat`

也就是說，背景並不是永遠只有顏色，而可能是：

- 純色背景
- 圖片背景
- 漸層背景

---

### `PPTAnimation`

動畫資料的欄位固定如下：

- `id`
- `elId`
- `effect`
- `type`
- `duration`
- `trigger`

其中 `trigger` 的值有：

- `click`
- `meantime`
- `auto`

這表示動畫除了效果本身，還會定義它：

- 作用在哪個元素
- 播放多久
- 什麼時候觸發

---

### `SlideType`

投影片類型的 union 值固定如下：

- `cover`
- `contents`
- `transition`
- `content`
- `end`

這些值不是單純的樣式，而是用來描述這頁在整份簡報中的語意角色。

---

## 最小例子：一頁投影片大概長這樣

```ts
const slide = {
  id: 'slide-1',
  elements: [
    {
      id: 'el-1',
      type: 'text',
      left: 120,
      top: 80,
      width: 300,
      height: 60,
      rotate: 0,
      content: '季度報告',
      defaultFontName: 'Microsoft YaHei',
      defaultColor: '#222222'
    }
  ],
  background: {
    type: 'solid',
    color: '#ffffff'
  },
  animations: [
    {
      id: 'ani-1',
      elId: 'el-1',
      effect: 'fadeIn',
      type: 'in',
      duration: 500,
      trigger: 'click'
    }
  ],
  sectionTag: {
    id: 'sec-1',
    title: '第一章'
  },
  type: 'cover'
}
```

這個例子要看懂的重點是：

- `elements` 是主內容
- `background`、`animations`、`sectionTag` 是頁面層級補充資料
- `type` 會說明這頁的角色

---

## 這一頁與 store 的關係

`slides store` 不只保存 `slides` 陣列，還會根據目前的狀態提供衍生資料，例如：

- `currentSlide`
- `currentSlideAnimations`
- `formatedAnimations`

所以更準確地說：

- `Slide` 是資料本體
- store 會再從 `Slide` 與 `slideIndex` 推導出目前頁面與動畫相關的衍生狀態
- `currentSlideAnimations` 會先把 `currentSlide.animations` 限定在目前頁面現有的元素 ID 範圍內
- `formatedAnimations` 會再依 `trigger` 把動畫整理成播放序列

這也是為什麼在閱讀原始碼時，要把「**資料本體**」和「**由 store 推導出的狀態**」分開看。

---

## 本篇重點整理

這一篇最重要的不是背欄位，而是搞懂下面三件事：

1. `SlidesState` 是整份簡報的狀態容器，不等於單頁內容
2. `Slide` 是單頁投影片的內容本體
3. `Slide` 除了 `elements` 之外，還會掛背景、動畫、註解、分段與頁面類型等頁面級資料

只要這三點清楚，後面去看 `PPTElement` 時就不容易迷路。

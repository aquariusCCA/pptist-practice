# 編輯器骨架總覽

這一章延續 [`02-編輯器骨架/README.md`](./README.md) 的脈絡，先把 PPTist 的編輯器相關入口與骨架看成一個整體。

這裡先不進到每個元件的細節，而是先回答三個問題：

1. `App.vue` 負責什麼
2. `Editor/index.vue` 負責什麼
3. `main store` 在這一層扮演什麼角色

如果先把這三層的邊界切清楚，後面再看 Canvas、Toolbar、快照、快捷鍵，就比較不會混在一起。

---

## 1. 這一章先看哪三個檔案

這一章的核心入口是：

1. [`src/App.vue`](/C:/Users/wtlc/Documents/pptist-practice/PPTist-SourceCode/src/App.vue)
2. [`src/views/Editor/index.vue`](/C:/Users/wtlc/Documents/pptist-practice/PPTist-SourceCode/src/views/Editor/index.vue)
3. [`src/store/main.ts`](/C:/Users/wtlc/Documents/pptist-practice/PPTist-SourceCode/src/store/main.ts)

它們的分工可以先這樣理解：

- `App.vue`：決定進哪個根視圖，並準備啟動資料
- `Editor/index.vue`：組裝桌面版編輯器骨架
- `main store`：承接 editor 需要的全域 UI 與操作狀態

## 2. 整體結構先看一眼

原始碼裡比較接近下面這個層級：

```text
App.vue
├─ Screen
├─ Editor
├─ Mobile
└─ FullscreenSpin

Editor/index.vue
├─ EditorHeader
├─ Thumbnails
├─ CanvasTool
├─ Canvas
├─ Remark
├─ Toolbar
├─ SelectPanel
├─ SearchPanel
├─ NotesPanel
├─ MarkupPanel
├─ SymbolPanel
├─ ExportDialog
└─ AIPPTDialog
```

這裡最重要的是分清楚：

- `App.vue` 只負責最外層根視圖切換
- `Editor/index.vue` 才是桌面編輯器的骨架組裝
- 彈窗、面板、工具列屬於 editor 內部

---

## 3. `App.vue` 是入口，不是 editor 骨架

`App.vue` 的 template 很單純，核心就是條件渲染：

- `slides.length` 有值時，才進入主要介面
- `screening` 為真時顯示 `Screen`
- 否則如果是桌面裝置，顯示 `Editor`
- 否則顯示 `Mobile`
- `slides.length` 還沒準備好時，顯示 `FullscreenSpin`

對照原始碼：

```vue
<template v-if="slides.length">
  <Screen v-if="screening" />
  <Editor v-else-if="_isPC" />
  <Mobile v-else />
</template>
<FullscreenSpin v-else />
```

所以 `App.vue` 的角色比較接近：

> **PPTist 的應用入口與根視圖分流器。**

它不是在組桌面版 editor，也不是在管理 Canvas 或工具列。

### `App.vue` 另外還做了兩件事

1. 進入頁面後，先載入 mock slides，放進 `slidesStore`
2. 建立 snapshot 資料庫，並清理上一輪留下的廢棄 DB 線索

這些事情的本質是「啟動前準備」，不是 editor 內部功能。

---

## 4. `Editor/index.vue` 才是桌面編輯器骨架

當 `App.vue` 決定進入 `Editor` 後，真正的桌面版骨架才開始。

`Editor/index.vue` 的 template 結構是：

- `EditorHeader`
- 左側 `Thumbnails`
- 中央區塊
  - `CanvasTool`
  - `Canvas`
  - `Remark`
- 右側 `Toolbar`
- 另外掛出 editor 內的面板與對話框

這表示 `Editor/index.vue` 的責任不是單一元件，而是把桌面編輯器的主要區塊拼起來。

### editor 內部的附屬面板

`Editor/index.vue` 還會根據 `main store` 狀態顯示這些面板與對話框：

- `SelectPanel`
- `SearchPanel`
- `NotesPanel`
- `MarkupPanel`
- `SymbolPanel`
- `ExportDialog`
- `AIPPTDialog`

這些都不是 `App.vue` 的責任。

---

## 5. `main store` 是這一層的全域狀態中心

`src/store/main.ts` 裡的 state 很多，但在這一章可以先抓幾個和 editor 骨架最直接相關的欄位：

- `showSelectPanel`
- `showSearchPanel`
- `showNotesPanel`
- `showSymbolPanel`
- `showMarkupPanel`
- `showAIPPTDialog`
- `dialogForExport`
- `toolbarState`

這些 state 的用途是讓 `Editor/index.vue` 能決定：

- 要不要顯示某個面板
- 要不要打開某個 modal
- 工具列目前處於哪種狀態

所以 `main store` 在這一層比較像：

> **支撐 editor 骨架運作的全域 UI 狀態容器。**

它不是入口分流本身，但它會影響 editor 內部的顯示與操作。

---

## 6. 這一章的閱讀順序

如果要依照原始碼順序整理，建議這樣讀：

1. `src/App.vue`
2. `src/views/Editor/index.vue`
3. `src/store/main.ts`

這個順序有兩個好處：

- 先看根入口怎麼決定進哪個模式
- 再看桌面 editor 骨架怎麼組起來，最後才看 state 怎麼驅動它

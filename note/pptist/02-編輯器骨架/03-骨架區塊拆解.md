# 骨架區塊拆解

這一篇把編輯器骨架中的固定區塊逐一拆開看。

你先把它當成「每一欄的責任表」，不要先急著背功能細節。

## 1. Header：整頁級操作入口

[`EditorHeader`](../../PPTist-SourceCode/src/views/Editor/EditorHeader/index.vue) 是整個編輯器最上方的控制列。

它主要做這幾件事：

- 顯示文件標題
- 開啟導入與匯出相關功能
- 進入放映模式
- 開啟 AI 生成 PPT 視窗
- 開啟快捷操作說明
- 提供 GitHub 與常見問題入口

### Header 的設計重點

- 左側偏向「文件與管理」
- 右側偏向「展示、匯出、外部連結」
- 它不是畫布操作工具列，而是更高一層的全局操作入口

### Header 與 store 的關係

它會直接操作：

- `dialogForExport`
- `showMarkupPanel`
- `showAIPPTDialog`
- `title`

所以 Header 是「頁面級控制」而不是單純 UI。

## 2. Thumbnails：頁面導航與節管理

[`Thumbnails`](../../PPTist-SourceCode/src/views/Editor/Thumbnails/index.vue) 是左側欄。

它的主要責任是：

- 顯示所有幻燈片縮圖
- 新增幻燈片
- 套用模板
- 切換幻燈片
- 支援多選頁面
- 支援拖曳排序
- 支援節（section）標題與管理

### Thumbnails 的關鍵行為

- 會跟 `slideIndex` 同步目前頁面
- 會根據 `selectedSlidesIndex` 顯示多選狀態
- 會在頁面切換時自動捲到目前縮圖
- 會透過 `vuedraggable` 做頁面排序

### Thumbnails 的重點

左側欄不是單純列表，它其實是：

- 頁面導航器
- 批次選取區
- 節管理區
- 頁面操作的右鍵入口

這就是它為什麼會跟 `useSlideHandler()`、`useSectionHandler()`、`useLoadSlides()` 一起工作。

## 3. CanvasTool：快速命令與插入工具

[`CanvasTool`](../../PPTist-SourceCode/src/views/Editor/CanvasTool/index.vue) 是中間上方那條工具列。

它集中放了三類操作：

### 左邊：歷史與輔助面板

- undo / redo
- 批註面板
- 选择窗格
- 查找替换

### 中間：插入內容

- 文字
- 形狀
- 圖片
- 線條
- 圖表
- 表格
- 公式
- 音視頻
- 符號

### 右邊：畫布縮放

- 縮小
- 顯示百分比
- 放大
- 適應屏幕

### CanvasTool 的意義

它把高頻操作集中在畫布上方，讓使用者不用切到右側工具欄才開始工作。

也就是說：

- Header 偏文件管理
- CanvasTool 偏快速操作
- Toolbar 偏屬性編輯

這三個層次是分開的。

## 4. Canvas：主編輯舞台

[`Canvas`](../../PPTist-SourceCode/src/views/Editor/Canvas/index.vue) 是這個骨架最重要的區塊，但它的細節要留到 `03-畫布系統`。

在骨架層，你只要先知道它做什麼：

- 承接元素的選取與操作
- 顯示主畫布內容
- 負責滑鼠與鍵盤在畫布上的互動入口
- 掛上對齊線、尺規、操作框、連結彈窗

### Canvas 在骨架中的位置

Canvas 不是整頁容器，它是中間工作區的核心區。

它前面有 `CanvasTool`，後面有 `Remark`，上面受 `EditorHeader` 控制，右邊受 `Toolbar` 影響。

## 5. Remark：講者備註區

[`Remark`](../../PPTist-SourceCode/src/views/Editor/Remark/index.vue) 是中間下方的備註欄。

它的工作很單純：

- 顯示目前頁面的 `remark`
- 允許編輯文字
- 支援拖曳改變高度
- 頁面切換時同步備註內容

### Remark 的設計重點

- 它不是獨立編輯器頁
- 它是頁面骨架的一部分
- 高度由父層控制，內容由 `slides store` 控制

## 6. Toolbar：右側屬性面板

[`Toolbar`](../../PPTist-SourceCode/src/views/Editor/Toolbar/index.vue) 是右側欄。

它會根據目前選取狀態切換不同面板：

- 沒選元素時，顯示頁面設計與切換
- 選到單一元素時，顯示樣式、位置、動畫
- 選到多個元素時，顯示多選樣式與多選位置

### Toolbar 的角色

它不是固定內容，而是「依狀態切換的屬性面板容器」。

這也是右側欄和左側欄最大的差異：

- 左側欄偏頁面
- 右側欄偏元素與頁面屬性

## 這一篇的結論

編輯器骨架其實就是這五個固定區塊：

- Header：全域入口
- Thumbnails：頁面導航
- CanvasTool：快速命令
- Canvas：主工作區
- Remark：講者備註
- Toolbar：屬性面板

先把這六塊的責任分清楚，後面你再看具體互動時，就不會把「框架」和「功能」混在一起。

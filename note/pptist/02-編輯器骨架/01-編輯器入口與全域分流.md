# 編輯器入口與全域分流

這一篇延續 [`02-編輯器骨架/README.md`](./README.md) 和 [`00-編輯器骨架總覽.md`](./00-編輯器骨架總覽.md) 的脈絡，先只看 `src/App.vue`。

這裡要處理的不是 editor 內部怎麼排版，而是更上層的問題：

> **PPTist 啟動後，第一個要顯示哪個根視圖，進入編輯器前又要先準備哪些資料。**

如果先把這一層看懂，後面再進 `Editor/index.vue`，就不容易把「入口分流」和「編輯器骨架」混在一起。

---

## 1. `App.vue` 先決定整個應用要顯示什麼

先看 `App.vue` 的 template。

它的邏輯很單純：

- `slides.length` 有值時，才進入主要介面
- `screening` 為真時，顯示 `Screen`
- 否則如果是桌面裝置，顯示 `Editor`
- 否則顯示 `Mobile`
- `slides.length` 還沒準備好時，顯示 `FullscreenSpin`

對應到原始碼就是這段：

```vue
<template v-if="slides.length">
  <Screen v-if="screening" />
  <Editor v-else-if="_isPC" />
  <Mobile v-else />
</template>
<FullscreenSpin v-else loading :mask="false" />
```

所以 `App.vue` 的角色不是 editor 骨架，而是：

> **PPTist 的最外層入口與根視圖分流器。**

這個專案也不是用 router 做這層切換，而是直接在 `App.vue` 以條件渲染決定顯示內容。

## 2. `App.vue` 也負責進入頁面時的資料準備

`App.vue` 的 `onMounted()` 會做三件事：

1. `api.getMockData('slides')` 讀入初始簡報資料
2. `slidesStore.setSlides(slides)` 把資料放進 `slides store`
3. `deleteDiscardedDB()` 清理廢棄的資料庫
4. `snapshotStore.initSnapshotDatabase()` 建立第一筆 snapshot

這段不是在做 editor 的 UI 組裝，而是在準備 editor 啟動所需的基礎資料。

### 2.1 `slidesStore` 接住簡報資料

`App.vue` 不會直接把簡報內容傳給 `Canvas` 或 `Thumbnails`，而是先交給 `slidesStore`。

這樣後面的元件只要從 store 讀資料，不需要知道入口是怎麼載入的。

### 2.2 `snapshotStore` 建立歷史基底

`snapshotStore.initSnapshotDatabase()` 會建立第一筆 snapshot，讓 undo / redo 有可回復的起點。

所以 `App.vue` 的啟動流程可以理解成：

- 先載入簡報
- 再清理舊 DB
- 最後建立 snapshot 基底

## 3. `App.vue` 還負責離開頁面時的廢棄 DB 標記

`App.vue` 另外有一段和資料生命週期有關的邏輯：

- 在非 development 環境下，先設定 `window.onbeforeunload = () => false`
- 再註冊 `beforeunload` 監聽
- 離開頁面時，把目前的 `databaseId` 記到 `localStorage`

這裡不是直接刪除 IndexedDB，而是先把這次使用過的 DB 標記成「待清理」。

真正的刪除發生在下一次啟動時，由 `deleteDiscardedDB()` 去處理。

這個流程的意義是：

- 讓舊資料庫不會一直堆積
- 把清理動作放在下一次啟動時統一處理
- 讓應用每次重新進來都能拿到比較乾淨的工作環境

## 5. 這一層不要和 `Editor/index.vue` 混在一起

`App.vue` 的責任是進入哪個模式，以及啟動前的準備工作。

真正的桌面編輯器骨架，是在 `src/views/Editor/index.vue` 才開始組裝：

- `EditorHeader`
- `Thumbnails`
- `CanvasTool`
- `Canvas`
- `Remark`
- `Toolbar`
- 各種面板與對話框

對應檔案：

- [`src/views/Editor/index.vue`](/C:/Users/wtlc/Documents/pptist-practice/PPTist-SourceCode/src/views/Editor/index.vue)

所以如果要用一句話切邊界，可以寫成：

> **`App.vue` 管入口，`Editor/index.vue` 管骨架。**

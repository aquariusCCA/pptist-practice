# 編輯器入口與全域分流

編輯器不是一打開就直接進到 `Editor`。

在進入編輯器之前，`src/App.vue` 先負責三件事：

1. 判斷要顯示哪個頂層模式
2. 把範例資料載入進來
3. 初始化快照資料庫與離開頁面的清理流程

## 1. App 先決定顯示哪個模式

[`src/App.vue`](../../PPTist-SourceCode/src/App.vue) 的 template 是這樣的概念：

- `slides.length` 有資料時才進入主畫面
- 如果正在放映，顯示 `Screen`
- 否則如果是桌機，顯示 `Editor`
- 其他情況顯示 `Mobile`
- 如果還沒載入資料，顯示全螢幕 loading

這代表 `App` 是一個「模式切換器」，不是編輯器內容本身。

## 2. App 的初始化流程

在 `onMounted()` 裡，`App.vue` 會：

- 透過 `api.getMockData('slides')` 讀入範例簡報
- 呼叫 `slidesStore.setSlides(slides)`
- 刪除被標記為廢棄的 IndexedDB
- 初始化 snapshot 資料庫

這裡的重點是：

- 編輯器畫面真正需要的資料，先由 `slides store` 接住
- 歷史快照相關機制，則交給 `snapshot store`

## 3. 離開頁面時的清理

`App.vue` 也有 `beforeunload` 的監聽：

- 先從 `localStorage` 取出被丟棄的資料庫清單
- 把目前 `databaseId` 加進去
- 再回寫到 `localStorage`

這個流程的用意是，離開頁面時不要留下無用的 IndexedDB 資料。

## 4. 編輯器入口的角色分工

`App.vue` 本身不處理編輯器操作細節，它只是提供入口。

真正的編輯器功能都會在 `src/views/Editor/index.vue` 之後展開，包括：

- 版面組裝
- 面板開關
- 快捷鍵與貼上事件
- 各區塊的協作

## 你要記住的結論

- `App.vue` 是模式入口
- `Editor` 是桌面編輯工作區
- 初始化資料和清理資料，都放在 `App.vue`
- 編輯器內部的互動，才是後面章節要看的重點

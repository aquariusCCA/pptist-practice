# 編輯器頁面骨架

`src/views/Editor/index.vue` 是整個桌面編輯器的骨架中心。

它不負責元素邏輯，也不負責畫布演算法；它負責把整個工作區擺正，讓其他元件有地方掛上去。

## 1. 版面結構

頁面可以拆成兩個層級：

### 第一層：主編輯器容器

`.pptist-editor` 佔滿整個高度。

### 第二層：頂部工具列 + 中間內容區

- `EditorHeader` 高度固定 `40px`
- `layout-content` 佔剩下的高度
- `layout-content` 使用 `flex`
- 左側 `Thumbnails` 固定 `160px`
- 右側 `Toolbar` 固定 `260px`
- 中間區域佔剩下的寬度

這種配置的好處是：

- 左右欄寬度穩定，方便做工具面板
- 中間主工作區會跟著視窗自動伸縮
- 編輯器骨架維持簡單、可預期

## 2. 中間工作區的垂直切分

中間欄再切成三塊：

- `CanvasTool`：上方工具列
- `Canvas`：主畫布
- `Remark`：下方批註區

其中 `Canvas` 的高度不是寫死的，而是用：

```vue
:style="{ height: `calc(100% - ${remarkHeight + 40}px)` }"
```

來扣掉上方 `CanvasTool` 的 `40px` 和下方 `Remark` 的高度。

這表示：

- 畫布會跟著批註區高度變動
- 批註區可以拖曳調整
- 中間區域的高度計算由頁面骨架負責，不由子元件自己猜

## 3. Remark 高度是骨架狀態

`remarkHeight` 是 `Editor/index.vue` 自己維護的本地狀態。

它的角色很單純：

- 初始值是 `40`
- 透過 `v-model:height` 交給 `Remark`
- `Remark` 改變高度後回寫到父層

也就是說，`Remark` 本身負責互動，父層負責整體布局同步。

## 4. 這一層還掛了哪些全域能力

在 `setup` 裡，`Editor/index.vue` 還直接啟用：

- `useGlobalHotkey()`
- `usePasteEvent()`

這代表它不是純視覺容器，而是編輯器工作區的全域事件入口。

## 5. 這一層還掛了哪些條件式浮層

主骨架下方另外掛了：

- `SelectPanel`
- `SearchPanel`
- `NotesPanel`
- `MarkupPanel`
- `SymbolPanel`
- `ExportDialog` 的 `Modal`
- `AIPPTDialog` 的 `Modal`

它們不是主版面的一部分，而是依照 `main store` 的布林狀態顯示或隱藏。

## 6. 這一層的設計重點

`Editor/index.vue` 的價值在於「集中組裝」：

- 排版在這裡定義
- 區塊在這裡掛載
- 輸入法、快捷鍵、貼上等全域事件在這裡啟用
- 其他互動細節都交給子元件

這種做法讓編輯器主結構很清楚，也讓後續功能能直接往對應區塊擴充。

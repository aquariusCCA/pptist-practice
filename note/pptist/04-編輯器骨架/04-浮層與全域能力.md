# 04-浮層與全域能力

這一篇接在 `03-骨架區塊拆解.md` 後面，專門處理 `Editor/index.vue` 骨架外層掛上的浮層面板、彈窗，以及 `setup` 裡啟用的全域能力。

前一篇已經把固定骨架拆完了，這一篇要補的是骨架外層的兩件事：

- 由 `main store` 控制的浮層面板與彈窗
- 由 `Editor/index.vue` 啟用的全域快捷鍵與貼上事件

這篇的重點不是「有哪些功能」，而是確認這些功能在原始碼裡是怎麼被掛上去、又是由誰控制。

---

## 1. 浮層掛載在骨架外層

在 `src/views/Editor/index.vue` 裡，固定骨架是這六塊：

- `EditorHeader`
- `Thumbnails`
- `CanvasTool`
- `Canvas`
- `Remark`
- `Toolbar`

而在這些固定骨架之外，還有一層浮層區塊被直接掛在 `Editor/index.vue`：

- `SelectPanel`
- `SearchPanel`
- `NotesPanel`
- `MarkupPanel`
- `SymbolPanel`
- `ExportDialog`
- `AIPPTDialog`

這個分工很重要：

- 固定骨架負責版面結構
- 浮層與彈窗負責額外流程

所以 `Editor/index.vue` 不只是排版，也是一個浮層掛載點。

## 2. 哪些浮層是由 `main store` 控制

這些浮層不是自己決定何時顯示，而是由 `main store` 提供狀態來源。

在 `Editor/index.vue` 裡，透過 `storeToRefs(mainStore)` 取出：

- `showSelectPanel`
- `showSearchPanel`
- `showNotesPanel`
- `showSymbolPanel`
- `showMarkupPanel`
- `showAIPPTDialog`
- `dialogForExport`

對應到 `src/store/main.ts`，這些狀態都真的存在，而且都有對應的 action：

- `setSelectPanelState`
- `setSearchPanelState`
- `setNotesPanelState`
- `setSymbolPanelState`
- `setMarkupPanelState`
- `setAIPPTDialogState`
- `setDialogForExport`

也就是說：

- `Editor/index.vue` 負責把浮層掛上去
- `main store` 負責提供顯示條件

## 3. 左側浮層面板是常駐掛載點

左側這五個面板是直接用 `v-if` 掛在 `Editor/index.vue` 上：

- `SelectPanel`
- `SearchPanel`
- `NotesPanel`
- `MarkupPanel`
- `SymbolPanel`

它們的特性是：

- 不是 `Modal`
- 不是獨立頁面
- 是編輯器內部的側邊浮層

這些面板的開啟與關閉，都由 `main store` 的布林狀態控制。

### 這裡可以特別注意

`CanvasTool` 和 `EditorHeader` 都會去切換這些面板的顯示狀態：

- `CanvasTool` 會切 `SelectPanel / SearchPanel / NotesPanel / SymbolPanel`
- `EditorHeader` 會切 `MarkupPanel`

所以這些浮層不是單一元件自己的私有狀態，而是編輯器骨架層共同控制的 UI。

## 4. `ExportDialog` 與 `AIPPTDialog` 是模態流程

這兩個不是側邊浮層，而是包在 `Modal` 裡的流程彈窗。

### `ExportDialog`

`ExportDialog` 的顯示條件是：

```ts
!!dialogForExport
```

這表示 `dialogForExport` 不是單純 boolean，而是 `DialogForExportTypes`。

在 `main store` 裡，它可以是像 `pptx`、`pdf` 這類導出類型；關閉時會被清成空字串：

```ts
mainStore.setDialogForExport('')
```

所以它的語意不是「開 / 關」而已，而是「目前正在開哪一種導出流程」。

### `AIPPTDialog`

`AIPPTDialog` 的顯示條件是：

```ts
showAIPPTDialog
```

而且它被設成：

- 不可點遮罩關閉
- 不可按 `Esc` 關閉
- 只能由明確動作關閉

這代表它也是一個明確的流程型彈窗，而不是一般可隨便關掉的浮層。

關閉時會呼叫：

```ts
mainStore.setAIPPTDialogState(false)
```

## 5. `Modal` 類浮層代表什麼

在這份原始碼裡，`Modal` 類浮層有一個共同特徵：

- 它承接的是獨立流程
- 它不是常駐工作區的一部分

`ExportDialog` 和 `AIPPTDialog` 都符合這個特性。

這種設計表示：

- 浮層面板偏向編輯器內部工具
- `Modal` 偏向需要中斷當前操作的獨立流程

這個區分如果寫準，就會更接近原始碼的實際結構。

## 6. `useGlobalHotkey()` 是 document 層級的快捷鍵監聽

`Editor/index.vue` 的 `setup` 會直接啟用：

- `useGlobalHotkey()`
- `usePasteEvent()`

其中 `useGlobalHotkey()` 不是綁在單一元件上的快捷鍵，而是全域 `document` 監聽。

### 它實際做了什麼

`useGlobalHotkey.ts` 在 `mounted` 時註冊：

- `document.addEventListener('keydown', keydownListener)`
- `document.addEventListener('keyup', keyupListener)`
- `window.addEventListener('blur', keyupListener)`

而且它會依據這些狀態決定能不能觸發：

- `editorAreaFocus`
- `thumbnailsFocus`
- `disableHotkeys`
- `ctrlKeyState`
- `shiftKeyState`
- `spaceKeyState`

### 它真的處理的是什麼

這個 hook 不只是熱鍵縮寫表，而是把整個編輯器常用行為串起來：

- 複製 / 剪下 / 貼上
- 多選
- undo / redo
- 刪除
- 元素移動
- 頁面切換
- 放映
- 畫布縮放
- 文字框 / 圖形 / 線條的快速建立
- 物件鎖定、群組、排序

所以在筆記裡，`useGlobalHotkey()` 最好不要只寫成「快捷鍵功能」，而要寫成「編輯器全域操作入口」。

## 7. `usePasteEvent()` 是貼上資料分流器

`usePasteEvent()` 也是在 `Editor/index.vue` 直接啟用的全域 hook，但它處理的不是鍵盤，而是 `paste` 事件。

### 它的觸發條件

這個 hook 只有在以下狀況才會處理貼上：

- `editorAreaFocus` 為真
- 或 `thumbnailsFocus` 為真
- `disableHotkeys` 沒有開啟

如果這些條件不成立，它會直接 return。

### 它實際處理什麼

`usePasteEvent.ts` 會先檢查 `clipboardData`，再分流成兩種情況：

- 影像貼上
- 文字貼上

#### 影像貼上

如果剪貼簿裡有 image file，就會轉成 dataURL，然後建立圖片元素。

#### 文字貼上

如果剪貼簿第一個項目是 `text/plain`，就會交給 `pasteTextClipboardData(text)`。

所以它的角色不是「貼上事件本身」，而是「依剪貼簿內容決定要貼成圖片還是文字」。

## 8. 這一層和骨架層的關係

如果把這一篇和前面幾篇合起來看，可以得到更完整的分工：

- `App.vue`：決定進哪個模式
- `Editor/index.vue`：組桌面編輯器骨架，並掛載浮層與全域能力
- `main store`：提供浮層顯示狀態
- `CanvasTool / EditorHeader`：切換浮層與啟動操作入口
- `useGlobalHotkey / usePasteEvent`：提供編輯器層級的全域行為

因此這一篇的重點不是再談一次「編輯器有哪些功能」，而是把這些功能的掛載位置和控制來源寫準。

---

## 9. 本篇結論

`Editor/index.vue` 的骨架外層，實際上同時承擔兩件事：

- 掛載浮層與流程彈窗
- 啟用全域快捷鍵與貼上事件

更精確地說：

- `SelectPanel / SearchPanel / NotesPanel / MarkupPanel / SymbolPanel` 是由 `main store` 控制的側邊浮層
- `ExportDialog / AIPPTDialog` 是包在 `Modal` 裡的獨立流程
- `useGlobalHotkey()` 是編輯器全域操作入口
- `usePasteEvent()` 是剪貼簿內容分流入口

如果要把這篇濃縮成一句話，可以寫成：

> `Editor/index.vue` 在骨架外層掛上浮層與流程彈窗，並啟用編輯器層級的快捷鍵與貼上能力；顯示條件主要由 `main store` 提供，行為入口則由 hooks 接手。

# 縮圖元件設計筆記

所屬章節：[04-縮圖系統](./README.md)

## 本節導讀

如果你已經理解 `03-畫布系統` 的固定邏輯尺寸與 `scale` 思路，這篇要回答的是另一個實作問題：`PPTist` 為什麼能用同一份投影片資料，同時畫出編輯器左側縮圖、播放頁縮圖列、手機預覽，甚至匯出用的高解析預覽。

這篇重點不在互動細節，而是在確認縮圖的渲染模型、重用策略，以及它和 `slides store` 之間的依賴關係。讀完後，再進 `05-store與狀態流` 會比較容易看懂哪些 state 是資料、哪些 state 是渲染契約。

## 關鍵字

- 主題：縮圖系統、縮圖元件、投影片預覽
- 別名：thumbnail、slide preview、小型渲染器
- 英文：`ThumbnailSlide`、`ThumbnailElement`
- 常見搜尋：`ThumbnailSlide`、`slidesLoadLimit`、`size / viewportRatio`、`target="thumbnail"`
- 易混淆：`size` 不是高度、縮圖不是另一套投影片資料、縮圖互動不等於縮圖渲染

## 建議回查情境

- 當你想確認縮圖是不是沿用主畫布座標系時
- 當你想搞懂 `size="120"` 和 `size="120 / viewportRatio"` 的差別時
- 當你想追 `ThumbnailSlide` 為什麼能被那麼多頁面共用時
- 當你想確認大量投影片時，縮圖怎麼避免一次全部渲染時

## 30 秒複習入口

- 這篇最核心的是：縮圖不是另一套資料或另一套座標系，它只是固定邏輯投影片的縮小渲染。
- 最容易混淆的是：`size` 在 `ThumbnailSlide` 裡代表寬度，外層高度是 `size * viewportRatio` 算出來的。
- 最常被用到的是：`scale = size / viewportSize`、`visible + slidesLoadLimit`、`ThumbnailElement` 重用既有元素基底元件。

## 速查區

### 核心定義

- `ThumbnailSlide`：把單張 `slide` 渲染成縮圖的共用元件
- `ThumbnailElement`：把 `slide.elements` 逐個映射到對應的基底元素元件
- `slidesLoadLimit`：限制目前要實際渲染幾張縮圖，避免一次吃掉太多成本

### 關鍵規則 / 公式 / 判準

```ts
outerWidth = size
outerHeight = size * viewportRatio

innerWidth = viewportSize
innerHeight = viewportSize * viewportRatio

scale = size / viewportSize
```

### 前提條件

- `slides store` 必須提供 `viewportSize` 與 `viewportRatio`
- 每張 `slide` 的元素座標仍然活在固定邏輯尺寸中
- 各場景容器只決定「我要多大的縮圖」與「這張縮圖要不要顯示」

### 常見錯誤

- 把 `size` 當成高度看
- 以為縮圖是另一套簡化資料
- 把「縮圖元件」和「縮圖欄的選取、拖曳、分組、右鍵選單」混成同一層責任

### 一句話對比

- `03-畫布系統`：解釋固定邏輯畫布怎麼映射到畫面
- `04-縮圖系統`：解釋同一套邏輯畫布怎麼被縮成多場景共用的 preview renderer

## 正文

### 1. 縮圖系統真正要解的問題

如果 `PPTist` 為每個場景都各寫一套「投影片縮圖畫法」，問題會很快變多：

- 編輯器左側縮圖一套
- 播放頁縮圖列一套
- 手機預覽一套
- 匯出前預覽一套

這樣會導致同一張投影片在不同地方看起來不一致，也讓元素型別一增加，就得同步改很多地方。

`PPTist` 的做法比較乾淨：  
先固定投影片的邏輯尺寸與元素資料，再做一個能接受不同顯示尺寸的 `ThumbnailSlide`。這樣不同場景只要決定「我需要多大的縮圖」，不用重新定義投影片怎麼畫。

### 2. `ThumbnailSlide` 本質上是縮小版的畫布

在 [ThumbnailSlide/index.vue](../../../PPTist-SourceCode/src/views/components/ThumbnailSlide/index.vue) 裡，可以看到它和主畫布沿用同一個套路：

- 外層 `.thumbnail-slide` 用 `size` 與 `viewportRatio` 決定實際顯示大小
- 內層 `.elements` 維持 `viewportSize x viewportSize * viewportRatio`
- 最後在內層套 `transform: scale(scale)`

這代表：

- 投影片元素的 `left/top/width/height` 仍然用邏輯座標
- 縮圖只是把整個邏輯世界縮小
- 不需要為縮圖重算一份新的元素資料

這和 `03-畫布系統` 的關係很直接：  
主畫布是把邏輯投影片映射到編輯區；縮圖則是把同一個邏輯投影片映射到更小的容器。

### 3. `size` 的真正語意是「縮圖寬度」

`ThumbnailSlide` 的縮放公式是：

```ts
scale = size / viewportSize
```

所以這裡的 `size` 不是高度，而是寬度。  
外層高度則是透過：

```ts
height = size * viewportRatio
```

算出來。

這可以解釋幾個看起來不一致、其實很一致的呼叫方式：

- 在 [Editor/Thumbnails/index.vue](../../../PPTist-SourceCode/src/views/Editor/Thumbnails/index.vue) 裡，直接傳 `size="120"`，意思是左側縮圖寬度固定 120px。
- 在 [PresenterView.vue](../../../PPTist-SourceCode/src/views/Screen/PresenterView.vue) 與其他播放頁縮圖列裡，會看到 `size="120 / viewportRatio"`。這不是怪寫法，而是呼叫端想固定「高度」約 120px，所以先把目標高度換回對應的寬度。
- 在 [ExportImage.vue](../../../PPTist-SourceCode/src/views/Editor/ExportDialog/ExportImage.vue) 裡，直接傳 `size="1600"`，表示這時候不是做小縮圖，而是把 `ThumbnailSlide` 拿來當高解析預覽與匯出的中介渲染器。

所以 `ThumbnailSlide` 的 API 雖然只有一個 `size`，但它其實足夠支撐：

- 固定寬度縮圖
- 固定高度縮圖
- 高解析預覽 / 匯出

### 4. `ThumbnailElement` 沒有重寫元素渲染，而是重用基底元件

縮圖系統真正省力的地方，不只在 `scale`，還在 `ThumbnailElement`。

在 [ThumbnailElement.vue](../../../PPTist-SourceCode/src/views/components/ThumbnailSlide/ThumbnailElement.vue) 裡，它做的事很單純：

1. 讀出 `elementInfo.type`
2. 把型別映射到對應的基底元件
3. 傳入 `elementInfo`
4. 額外帶上 `target="thumbnail"`

對應關係像這樣：

- 圖片 → `BaseImageElement`
- 文字 → `BaseTextElement`
- 形狀 → `BaseShapeElement`
- 線條 → `BaseLineElement`
- 圖表 → `BaseChartElement`
- 表格 → `BaseTableElement`
- LaTeX → `BaseLatexElement`
- 影片 → `BaseVideoElement`
- 音訊 → `BaseAudioElement`

也就是說，縮圖不是另外寫一套「縮圖專用文字元件」或「縮圖專用圖片元件」，而是直接沿用現有的基底渲染元件。這讓縮圖和正式畫布更容易保持一致。

另外，`target="thumbnail"` 也提供了一個場景開關。例如：

- [BaseTextElement.vue](../../../PPTist-SourceCode/src/views/components/element/TextElement/BaseTextElement.vue) 會在 `target === 'thumbnail'` 時，把文字內容加上 `thumbnail` class，關掉 `pointer-events`
- [BaseChartElement.vue](../../../PPTist-SourceCode/src/views/components/element/ChartElement/BaseChartElement.vue) 也會在縮圖場景下加上 `is-thumbnail` class，避免圖表在縮圖裡接管互動

這說明縮圖系統的方向不是「重寫一份更簡陋的 DOM」，而是「在同一套元素基底上，對縮圖場景做必要的互動降級」。

### 5. `visible` 和 `useLoadSlides` 是縮圖效能的第一道保護

如果投影片很多，每張縮圖都立刻完整渲染，成本會很高。`PPTist` 這裡做了兩層保護：

第一層在 [useLoadSlides.ts](../../../PPTist-SourceCode/src/hooks/useLoadSlides.ts)：

- 初始 `slidesLoadLimit = 50`
- 如果投影片數量超過上限，每隔 `600ms` 再多開 `20`
- 若數量已經足夠，就直接把上限設成 `9999`

第二層在 `ThumbnailSlide` 自己：

- `visible` 預設是 `true`
- 如果 `visible` 為 `false`，只顯示 placeholder，不渲染 `.elements`

所以各個列表容器常見的寫法會是：

```vue
:visible="index < slidesLoadLimit"
```

這代表縮圖系統不是用真正的虛擬清單，而是用「分批打開渲染」的策略，先把前面幾張畫出來，再逐步補後面。

### 6. `ThumbnailSlide` 是一個跨場景的共用 renderer

目前讀到的原始碼裡，`ThumbnailSlide` 至少被這些場景共用：

- 編輯器左側縮圖欄：[Editor/Thumbnails/index.vue](../../../PPTist-SourceCode/src/views/Editor/Thumbnails/index.vue)
- 播放頁縮圖視圖：[Screen/SlideThumbnails.vue](../../../PPTist-SourceCode/src/views/Screen/SlideThumbnails.vue)
- 演講者模式底部縮圖列：[Screen/PresenterView.vue](../../../PPTist-SourceCode/src/views/Screen/PresenterView.vue)
- 行動版預覽：[Mobile/MobilePreview.vue](../../../PPTist-SourceCode/src/views/Mobile/MobilePreview.vue)
- 模板挑選：[Editor/Thumbnails/Templates.vue](../../../PPTist-SourceCode/src/views/Editor/Thumbnails/Templates.vue)
- 投影片連結預覽：[Editor/Canvas/LinkDialog.vue](../../../PPTist-SourceCode/src/views/Editor/Canvas/LinkDialog.vue)
- 匯出圖片前的渲染載體：[Editor/ExportDialog/ExportImage.vue](../../../PPTist-SourceCode/src/views/Editor/ExportDialog/ExportImage.vue)

這裡很值得注意的一點是：  
不同場景共用的是「投影片怎麼被畫出來」，但不共用「外層互動」。例如：

- 編輯器左側縮圖欄自己處理選取、拖曳排序、section 標題、右鍵選單
- 播放頁自己處理切頁與 active 狀態
- 手機預覽自己處理滿版寬度與底部操作列
- 匯出流程只拿它當純渲染器，不需要互動

所以責任切分很清楚：

- `ThumbnailSlide`：負責畫
- 外層容器：負責場景互動與版面控制

### 7. 這章是通往 `05-store與狀態流` 的橋

縮圖系統表面上在講元件，其實已經碰到兩種 store 角色：

#### `slides store` 提供的是渲染契約

在 [slides.ts](../../../PPTist-SourceCode/src/store/slides.ts) 裡，縮圖會直接依賴：

- `slides`
- `slideIndex`
- `viewportSize`
- `viewportRatio`

這些欄位不是單純「資料放哪裡」，而是整個畫面渲染的共同契約。  
一旦 `viewportSize` 或 `viewportRatio` 變了，主畫布、縮圖、播放頁就會一起改變。

#### `main store` 提供的是場景互動狀態

在 [Editor/Thumbnails/index.vue](../../../PPTist-SourceCode/src/views/Editor/Thumbnails/index.vue) 裡，縮圖欄的互動會再依賴：

- `selectedSlidesIndex`
- `thumbnailsFocus`
- `activeElementIdList`
- `disableHotkeys`

這些 state 不是 `ThumbnailSlide` 本身必需，但它們決定了縮圖欄怎麼操作、怎麼選、多選後怎麼呈現。

所以讀完這章後，進 `05-store與狀態流` 時可以帶著一個更清楚的問題去看：

> 哪些 state 是投影片本體的資料契約，哪些 state 是某個場景的操作狀態？

### 8. 目前能確認、推論與保留的地方

#### 已確認

- `ThumbnailSlide` 使用和主畫布同型的「固定邏輯尺寸 + 外層顯示尺寸 + 內層 `scale()`」模型
- `size` 在 `ThumbnailSlide` 裡代表寬度
- `ThumbnailElement` 會重用既有元素基底元件，而不是再做一套縮圖專用元件
- `visible + slidesLoadLimit` 是目前縮圖列表避免一次渲染過多頁的主要策略
- `ThumbnailSlide` 確實被多個場景共用，不只左側縮圖欄

#### 合理推論

- `ThumbnailSlide` 也會 `provide(injectKeySlideScale, scale)`，和主畫布、播放頁維持同一種 slide-scale 注入介面
- 但就目前讀到的縮圖渲染鏈，沒有看到縮圖子元件直接 `inject` 這個值，因此它更像是一個保留的統一介面，而不是目前縮圖場景的必要依賴

#### 待驗證

- 若之後要追更細的多媒體縮圖呈現差異，還可以再往影片、音訊、圖表等個別元素的場景特化邏輯補讀

## 附錄：`ThumbnailSlide` 原始碼

原始檔案：[src/views/components/ThumbnailSlide/index.vue](../../../PPTist-SourceCode/src/views/components/ThumbnailSlide/index.vue)

```vue
<template>
  <div class="thumbnail-slide"
    :style="{
      width: size + 'px',
      height: size * viewportRatio + 'px',
    }"
  >
    <div 
      class="elements"
      :style="{
        width: viewportSize + 'px',
        height: viewportSize * viewportRatio + 'px',
        transform: `scale(${scale})`,
      }"
      v-if="visible"
    >
      <div class="background" :style="backgroundStyle"></div>
      <ThumbnailElement
        v-for="(element, index) in slide.elements"
        :key="element.id"
        :elementInfo="element"
        :elementIndex="index + 1"
      />
    </div>
    <div class="placeholder" v-else>加载中...</div>
  </div>
</template>

<script lang="ts" setup>
import { computed, provide } from 'vue'
import { storeToRefs } from 'pinia'
import { useSlidesStore } from '@/store'
import type { Slide } from '@/types/slides'
import { injectKeySlideScale } from '@/types/injectKey'
import useSlideBackgroundStyle from '@/hooks/useSlideBackgroundStyle'

import ThumbnailElement from './ThumbnailElement.vue'

const props = withDefaults(defineProps<{
  slide: Slide
  size: number
  visible?: boolean
}>(), {
  visible: true,
})

const { viewportRatio, viewportSize } = storeToRefs(useSlidesStore())

const background = computed(() => props.slide.background)
const { backgroundStyle } = useSlideBackgroundStyle(background)

const scale = computed(() => props.size / viewportSize.value)
provide(injectKeySlideScale, scale)
</script>

<style lang="scss" scoped>
.thumbnail-slide {
  background-color: #fff;
  overflow: hidden;
  user-select: none;
}
.elements {
  transform-origin: 0 0;
}
.background {
  width: 100%;
  height: 100%;
  background-position: center;
  position: absolute;
}
.placeholder {
  width: 100%;
  height: 100%;
  display: flex;
  justify-content: center;
  align-items: center;
}
</style>
```

## 附錄：`ThumbnailElement` 原始碼

原始檔案：[src/views/components/ThumbnailSlide/ThumbnailElement.vue](../../../PPTist-SourceCode/src/views/components/ThumbnailSlide/ThumbnailElement.vue)

```vue
<template>
  <div 
    class="base-element"
    :class="`base-element-${elementInfo.id}`"
    :style="{
      zIndex: elementIndex,
    }"
  >
    <component
      :is="currentElementComponent"
      :elementInfo="elementInfo"
      target="thumbnail"
    ></component>
  </div>
</template>

<script lang="ts" setup>
import { computed } from 'vue'
import { ElementTypes, type PPTElement } from '@/types/slides'

import BaseImageElement from '@/views/components/element/ImageElement/BaseImageElement.vue'
import BaseTextElement from '@/views/components/element/TextElement/BaseTextElement.vue'
import BaseShapeElement from '@/views/components/element/ShapeElement/BaseShapeElement.vue'
import BaseLineElement from '@/views/components/element/LineElement/BaseLineElement.vue'
import BaseChartElement from '@/views/components/element/ChartElement/BaseChartElement.vue'
import BaseTableElement from '@/views/components/element/TableElement/BaseTableElement.vue'
import BaseLatexElement from '@/views/components/element/LatexElement/BaseLatexElement.vue'
import BaseVideoElement from '@/views/components/element/VideoElement/BaseVideoElement.vue'
import BaseAudioElement from '@/views/components/element/AudioElement/BaseAudioElement.vue'

const props = defineProps<{
  elementInfo: PPTElement
  elementIndex: number
}>()

const currentElementComponent = computed<unknown>(() => {
  const elementTypeMap = {
    [ElementTypes.IMAGE]: BaseImageElement,
    [ElementTypes.TEXT]: BaseTextElement,
    [ElementTypes.SHAPE]: BaseShapeElement,
    [ElementTypes.LINE]: BaseLineElement,
    [ElementTypes.CHART]: BaseChartElement,
    [ElementTypes.TABLE]: BaseTableElement,
    [ElementTypes.LATEX]: BaseLatexElement,
    [ElementTypes.VIDEO]: BaseVideoElement,
    [ElementTypes.AUDIO]: BaseAudioElement,
  }
  return elementTypeMap[props.elementInfo.type] || null
})
</script>
```

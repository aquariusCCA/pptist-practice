# `zoomAroundPoint()` 超直觀說明（強化版）
## 為什麼縮放時，滑鼠指到的點可以被「釘住」？

這份說明只回答一件事：

> 為什麼縮放時，明明內容變大或變小了，滑鼠底下原本指到的那個點，還能留在原位？

這篇會保留示意圖，而且把幾個最容易卡住的地方補完整：

- 到底有哪幾種座標？
- 為什麼單純改 `zoom`，點一定會跑掉？
- 為什麼一定要再補 `pan`？
- 公式到底怎麼一步一步推到程式碼？
- 縮小時也成立嗎？

---

# 0. 先講結論

`zoomAroundPoint()` 之所以能把點「釘住」，不是因為它做了什麼神秘魔法，而是因為它**強制滿足同一個條件**：

> 縮放前滑鼠底下對應的是哪個內容點，縮放後還是同一個內容點。

也就是說，它固定的不是「滑鼠座標本身」，而是：

- 滑鼠底下對應到的內容點
- 也就是同一個 `world point` / `logical point`

你可以把整件事濃縮成三句話：

```text
1. 先找出滑鼠底下對應的是哪個內容點
2. 單純縮放後，這個點理論上會跑掉
3. 所以再算一個新的 pan，把它拉回滑鼠底下
```

這就是 `zoomAroundPoint()` 的本質。

---

# 1. 先把三種座標講清楚

這是很多人第一次看這類公式時最容易混亂的地方。

其實你可以把它拆成三種座標：

## (1) DOM / Stage 座標

先不要急著想 world、pan、zoom，先只看一件事：

> **滑鼠現在落在 stage 這塊矩形區域的哪個位置？**

這個位置，就是 `anchor`。

而 `anchor` 使用的是：

> **以 stage 左上角為原點 `(0, 0)` 的座標系**

也就是說，`anchor.x` 表示：

- 從 **stage 左邊界** 往右數了多少距離

`anchor.y` 表示：

- 從 **stage 上邊界** 往下數了多少距離

示意圖：

```text
DOM / Stage 座標（左上角為原點）

                x →
    (0,0) ●-----------------------------------------+
          |                                         |
          |                                         |
          |                         ● anchor        |
          |                        (x, y)           |
          |                                         |
          |                ● center                 |
          |                                         |
          +-----------------------------------------+
                         ↓
                         y
````

你可以把它想成：

* `anchor.x` = 滑鼠距離 **stage 左邊** 多遠
* `anchor.y` = 滑鼠距離 **stage 上邊** 多遠

例如：

```text
stage 寬 800，高 600
滑鼠落在距離左邊 500、距離上邊 180 的地方
```

那就是：

```ts
anchor = { x: 500, y: 180 }
```

---

### 為什麼要特別強調「相對 stage」？

因為滑鼠事件一開始拿到的，常常不是 stage 內部座標，而是瀏覽器頁面座標。

例如滑鼠事件可能先給你：

* `clientX`
* `clientY`

但這兩個值通常是：

> **相對整個視窗（viewport）**

不是相對 stage。

所以如果你的 stage 並不貼在視窗左上角，那就不能直接把 `clientX` / `clientY` 當成 `anchor.x` / `anchor.y` 使用。

要先扣掉 stage 自己在頁面中的偏移量。

示意圖：

```text
瀏覽器視窗

(0,0) ●
       \
        \
         \        +----------------------------+
          \       |          stage             |
           \      |                            |
            \     |               ● 滑鼠       |
             \    |                            |
              \   +----------------------------+
               \
                這段距離如果不扣掉，
                就不是 stage 內部座標
```

也就是說：

```text
anchor = 滑鼠在 stage 內部的位置
```

不是：

```text
anchor = 滑鼠在整個瀏覽器視窗的位置
```

---

## (2) 以畫面中心為原點的座標

前一節的 `anchor`，表示的是：

> **滑鼠在 stage 裡的哪個位置**
> （以左上角為原點）

但很多 canvas / editor 的縮放、拖曳、旋轉邏輯，會更習慣用：

> **畫面中心當作參考基準**

因為這樣比較容易看出：

- 滑鼠是在中心的左邊還是右邊
- 滑鼠是在中心的上方還是下方
- 滑鼠離中心有多遠

所以，程式通常會先把滑鼠座標從：

```text
左上角原點
```

改寫成：

```text
以畫面中心為原點
```

程式碼就是這一步：

```ts
const anchorOffsetX = anchor.x - stageCenterX
const anchorOffsetY = anchor.y - stageCenterY
```

意思是：

> 把滑鼠位置改寫成「相對畫面中心偏移了多少」。

---

### 圖上怎麼看？

```text
DOM / Stage 座標（原本是左上角為原點）

+--------------------------------------------------+
|                                                  |
|                ● center                          |
|                         \                        |
|                          \                       |
|                           ● anchor               |
|                                                  |
+--------------------------------------------------+

anchorOffset = anchor - center
```

這個式子的意思不是在算神秘公式，而是在算：

> **滑鼠相對中心，多了多少位移**

---

### 用 X 軸單獨看最簡單

假設：

* stage 寬度是 `800`
* 那畫面中心 `stageCenterX = 400`
* 滑鼠現在在 `anchor.x = 700`

那就會得到：

```ts
anchorOffsetX = 700 - 400 = 300
```

這個 `300` 的意思不是：

> 滑鼠在整個畫面中的 X 是 300

而是：

> **滑鼠在畫面中心的右邊 300**

示意圖：

```text
X 軸示意

左邊界                 中心                           右邊界
  0 ------------------- 400 -------------------------- 800
                         |-------------- 300 ---------->|
                                                     anchor
```

所以：

```text
anchorOffsetX = 300
```

真正代表的是：

> 滑鼠相對中心，往右偏了 300

如果滑鼠在中心左邊，例如 `anchor.x = 250`，那：

```ts
anchorOffsetX = 250 - 400 = -150
```

示意圖：

```text
X 軸示意

左邊界                        中心                          右邊界
  0 ------------------------- 400 ------------------------- 800
  anchor <---- 150 -----------|
```

這時：

```text
anchorOffsetX = -150
```

意思就是：

> 滑鼠在中心左邊 150

---

### 為什麼這一步很重要？

因為後面我們要處理縮放時，常常不是在想：

> 滑鼠離左上角多遠

而是在想：

> 滑鼠相對畫面中心在哪裡

這樣後面把 `pan`、`zoom`、內容位置串起來時，會更直觀。

---

## (3) world 座標（內容自己的座標）

前面講的 `anchor`、`anchorOffset`，都是：

> **滑鼠在畫面上的位置**

但現在要講的 `world` 不一樣。

`world` 不是滑鼠位置，  
而是：

> **內容本身某個點，在內容世界裡的座標**

也可以理解成：

> **內容自己的邏輯座標**

---

### 先把兩者硬分開

`anchor` 是畫面上的點。  
`world` 是內容裡的點。

這兩個最容易混淆，但一定要分清楚：

```text
anchor = 滑鼠現在在畫面哪裡
world  = 某個內容點本來在內容世界的哪裡
```

---

### 用一個最簡單的例子理解

假設你的畫布內容上有一個點 `P`。

這個點 `P` 在內容世界裡的位置是：

```text
world = 200
```

這句話的意思是：

> `P` 在內容自己的座標系中，位於第 200 的位置

它不是在說：

> `P` 現在畫在螢幕上的 200 位置

因為內容最後畫到螢幕上時，還會受到：

* `pan`（平移）
* `zoom`（縮放）

影響。

所以同一個 `world = 200`，畫到畫面上時，位置可能完全不同。

---

### 示意圖：內容座標 vs 畫面座標

先看內容世界裡：

```text
內容世界（world）

0 ------------------- 100 ------------------- 200 -------------------→
                                                ● P
```

這表示：

> `P` 在內容裡固定就是第 200 的位置

但它顯示到畫面上時，不一定還在 200。

例如如果有平移和縮放，畫面上可能變成這樣：

```text
畫面上（screen）

0 ------------------- 100 ------------------- 300 -------------------→
                                                ● P
```

所以你要記住：

> `world = 200` 是內容自己的位置
> `screen = 300` 是它最後顯示在畫面上的位置

這是兩個不同層次的座標。

---

### 為什麼一定要有 world 座標？

因為我們要做到的事不是：

> 縮放後讓畫面上的某個像素留住

而是：

> 縮放前滑鼠底下對應到哪個內容點，縮放後還是同一個內容點

這裡的「同一個內容點」，指的就是同一個 `world` 點。

也就是說，`zoomAroundPoint()` 真正想釘住的，不是單純的畫面位置，而是：

> **滑鼠底下對應到的那個 world point**

---

### 這一節你只要先記一句話

```text
anchor 是畫面座標
world 是內容座標
```

後面公式要做的事，就是把兩者對起來。

---

# 2. 這份推導的前提是什麼？

在開始推公式之前，要先講清楚一件很重要的事：

> **這份推導裡，`screen` 不是「左上角為原點」的畫面座標。**
>  
> 它用的是：
> **以畫面中心為原點的座標系。**

這個前提非常重要，因為如果座標基準不同，後面的公式寫法也會不同。

---

## 先講結論

這份說明採用的是這種變換關係：

```text
screen = pan + world * zoom
```

但這裡的 `screen`，指的不是：

```text
相對 stage 左上角的座標
```

而是：

```text
相對畫面中心的座標
```

所以這個公式其實更精確地說，是：

> **內容點經過縮放與平移後，落在「以中心為原點」的畫面位置**

---

## 為什麼這件事很重要？

因為滑鼠一開始拿到的 `anchor`，通常是：

> **相對 stage 左上角的座標**

示意圖：

```text
Stage 座標（左上角原點）

+--------------------------------------------------+
| (0,0)                                            |
|   ●                                              |
|                                                  |
|                ● center                          |
|                         ● anchor                 |
|                                                  |
+--------------------------------------------------+
```

但後面公式裡要使用的 `screen`，卻是：

> **相對畫面中心的座標**

也就是這種感覺：

```text
以畫面中心為原點

                        y
                        ↑
                        |
              (-)       |       (+)
------------------------●------------------------→ x
                      center
                           \
                            \
                             ● anchorOffset
```

所以，這兩者其實不是同一種座標。

如果你直接把 `anchor` 拿去代進：

```text
screen = pan + world * zoom
```

座標系就會不一致，推導會混亂。

---

## 所以一定要先做這一步

```ts
const anchorOffsetX = anchor.x - stageCenterX
const anchorOffsetY = anchor.y - stageCenterY
```

這一步的目的就是：

> **把滑鼠位置從「左上角原點座標」**
> 轉成
> **「以畫面中心為原點的座標」**

也就是：

```text
anchorOffset = anchor - center
```

轉完之後，滑鼠位置和公式裡的 `screen` 就站在同一個座標系上了。

這樣你後面才可以正確地說：

> 滑鼠底下對應到哪個 `world` 點

以及：

> 縮放後，怎麼調整 `pan` 才能讓同一個 `world` 點還留在滑鼠底下

---

## 用一句話記住這個前提

這份推導不是直接拿「左上角座標」來算，
而是先把所有畫面位置都統一改寫成：

> **相對畫面中心的座標**

所以才要先做：

```ts
anchorOffset = anchor - center
```

然後才接：

```text
screen = pan + world * zoom
```
---


# 3. 先建立直覺：為什麼點會跑掉？

先只看 X 軸，Y 軸完全同理。

先假設目前使用的是：

> **以畫面中心為原點的座標系**

並且先用最簡單的情況來理解：

- `pan = 0`
- 滑鼠現在在畫面右邊 `300`
- 內容裡有一個點 `P`
- 這個點 `P` 此時**剛好顯示在畫面上的 `300` 位置**

也就是說：

> 現在滑鼠剛好正指著內容點 `P`

---

## 縮放前

```text
以畫面中心為原點

←------------------- 0 -------------------→
                    center

                              ↑ 滑鼠
                              |
------------------------------P------------------------------
                             300
```

此時：

* 滑鼠在畫面上的位置是 `300`
* 內容點 `P` 顯示在畫面上的位置也是 `300`

所以滑鼠正好指著 `P`。

---

## 只放大，不調整 pan

現在假設把內容放大 2 倍，但完全不改 `pan`。

因為是以中心為原點，所以：

> 原本離中心 `300` 的點，放大 2 倍後，會跑到離中心 `600`

也就是：

```text
300 × 2 = 600
```

圖上會變成：

```text
放大後，內容被拉開

←------------------- 0 -------------------→
                    center

                              ↑ 滑鼠
                              |
------------------------------------------P------------------
                                         600
```

你會發現：

* **滑鼠在畫面上的位置沒變**
* 但內容點 `P` 被放大效果往外推開了
* 所以滑鼠原本指著的那個點，現在不在滑鼠底下了

---

## 這就是縮放時的根本問題

`zoom` 做的事情只有一個：

> 讓內容相對中心放大或縮小

它不會自動幫你保證：

> 原本滑鼠底下的那個內容點，縮放後還留在原位

所以如果你希望：

> 縮放前滑鼠指到哪個點，縮放後還是同一個點

那就不能只改 `zoom`，還必須**額外調整 `pan`**，把跑掉的點拉回來。

---

# 4. `pan` 的工作到底是什麼？

`pan` 的工作其實非常單純：

> **把因為縮放而跑掉的內容，整體往反方向拉回來。**

注意，這裡不是只移動某一個點 `P`，而是：

> **把整個內容一起平移。**

因為 `pan` 本質上做的事，就是整體平移。

---

## 先回顧剛才發生了什麼

剛才的例子裡：

- 原本內容點 `P` 顯示在畫面上的 `300`
- 放大 2 倍後，如果不調整 `pan`
- `P` 會跑到 `600`

圖上是這樣：

```text
只做 zoom，不補 pan

←------------------- 0 -------------------→
                    center

                              ↑ 滑鼠
                              |
------------------------------------------P------------------
                                         600
```

但滑鼠仍然停在畫面上的 `300`。

所以現在的問題變成：

> 我們要怎麼把跑到 `600` 的 `P`，重新拉回滑鼠所在的 `300`？

---

## 做法：用 pan 把整個內容往左拉

因為 `P` 現在在 `600`，
而我們希望它回到 `300`，
所以要把整個內容往左平移 `300`。

也就是：

```text 
600 - 300 = 300
```

示意圖可以這樣看：

```text 
補 pan 前

←------------------- 0 -------------------→
                    center

                              ↑ 滑鼠
                              |
------------------------------------------P------------------
                                         600
```

```text
整個內容往左拉 300
<--------------------------- 300 ---------------------------->
```

```text
補 pan 後

←------------------- 0 -------------------→
                    center

                              ↑ 滑鼠
                              |
------------------------------P------------------------------
                             300
```

這樣 `P` 就重新回到滑鼠底下了。

---

## 所以 `pan` 真正做的事是什麼？

`pan` 不是在幫你「改變縮放倍率」，
而是在幫你做這件事：

> **補償縮放造成的位置偏移**

也就是說：

* `zoom` 會讓內容相對中心放大或縮小
* `pan` 會把因縮放而跑掉的位置補回來

---

## 整件事其實就是兩步

```text
1. zoom 讓點跑掉
2. pan 把整個內容拉回來
```

或者你也可以寫得更精確一點：

```text
1. zoom 改變內容和中心之間的距離
2. pan 補償這個距離變化，讓指定的點回到原位
```

---

## 用一句話記住這一節

`zoom` 負責放大縮小，
`pan` 負責把滑鼠想釘住的那個點補回來。

---

# 5. 程式碼第一步到底在做什麼？

這段程式碼做的事情其實很單純：

> **先找出畫面中心在哪裡，再把滑鼠位置改寫成「相對中心」的座標。**

因為後面我們要使用的變換關係是：

```text
screen = pan + world * zoom
```

而這裡的 `screen`，是：

> **以畫面中心為原點的座標**

所以滑鼠位置也必須先轉到同一個座標系裡。

---

## 第一步：先找出畫面中心

```ts
const stageCenterX = stageSize.width / 2
const stageCenterY = stageSize.height / 2
```

這一步不是在做複雜運算，
只是先找出：

> **這個 stage 的正中央在哪裡**

例如：

* `stageSize.width = 800`
* `stageSize.height = 600`

那中心點就是：

```ts
stageCenterX = 400
stageCenterY = 300
```

圖：

```text
Stage（左上角為原點）

+--------------------------------------------------+
|                                                  |
|                                                  |
|                     ● center                     |
|                   (400, 300)                     |
|                                                  |
+--------------------------------------------------+
```

這一步的目的只有一個：

> 後面要把滑鼠座標改寫成「相對中心」，所以要先知道中心的位置。

---

## 第二步：把滑鼠改寫成「相對中心」座標

```ts
const anchorOffsetX = anchor.x - stageCenterX
const anchorOffsetY = anchor.y - stageCenterY
```

這一步的重點不是在算距離公式，
而是在做一件事：

> **把滑鼠位置從「左上角原點」的表示方式，改成「以中心為原點」的表示方式。**

也就是：

```text
anchorOffset = anchor - center
```

這裡的減法意思是：

> 滑鼠相對畫面中心，往右或往左偏了多少
> 滑鼠相對畫面中心，往下或往上偏了多少

---

## 圖上怎麼理解？

```text
Stage（左上角原點）

+--------------------------------------------------+
|                                                  |
|                     ● center                     |
|                           \                      |
|                            \                     |
|                             ● anchor             |
|                                                  |
+--------------------------------------------------+

anchorOffset = anchor - center
```

看圖時，你可以把它理解成：

* `center` 是新的參考基準
* `anchorOffset` 是滑鼠相對這個基準的位移

---

## 用數字看一次最清楚

假設：

```ts
stageCenterX = 400
anchor.x = 700
```

那就得到：

```ts
anchorOffsetX = 700 - 400 = 300
```

這個 `300` 的意思不是：

> 滑鼠的 X 座標是 300

而是：

> **滑鼠在畫面中心的右邊 300**

如果滑鼠在中心左邊，例如：

```ts
anchor.x = 250
```

那就會得到：

```ts
anchorOffsetX = 250 - 400 = -150
```

這時代表：

> **滑鼠在畫面中心的左邊 150**

示意圖：

```text
X 軸示意

左邊界                 中心                           右邊界
  0 ------------------- 400 -------------------------- 800
                         |-------------- 300 ---------->|
                                                     anchor
```

---

## 為什麼這一步很重要？

因為後面不管是：

* `pan`
* `world * zoom`

都是在同一個「以畫面中心為基準」的座標系裡理解的。

所以滑鼠位置也一定要先轉到同一套座標系中，
後面才能正確推導：

* 滑鼠底下對應哪個 `world` 點
* 縮放後要怎麼調整 `pan`
* 才能把同一個點釘在滑鼠底下

---

# 6. 真正的核心公式：螢幕位置怎麼來的？

前面一直在鋪陳，現在終於來到真正的核心公式：

```text
screen = pan + world * zoom
```

這個公式的意思是：

> **內容中的一個點，先經過縮放，再經過整體平移，最後才出現在畫面上的某個位置。**

---

## 先強調一個前提

這裡的 `screen` 不是：

```text
相對左上角原點的畫面座標
```

而是：

```text
相對畫面中心的畫面座標
```

也就是說，這個公式裡討論的所有位置，都是站在：

> **以畫面中心為原點**

這套座標系裡看的。

---

## 公式裡每一項代表什麼？

```text
screen = pan + world * zoom
```

可以拆成三個部分理解：

* `world`：內容點原本在內容世界裡的位置
* `world * zoom`：這個點經過縮放後，離原點的距離被放大或縮小
* `pan`：再把整個內容整體平移
* `screen`：最後這個點顯示在畫面上的位置

---

## 用圖理解這個公式

先看「縮放後，但還沒平移」的狀態：

```text
以畫面中心為原點

0 -------------------------- world * zoom --------------------------→
                               ●
```

這表示：

> 內容點經過縮放後，會先落在某個相對中心的位置

接著，再把整個內容做平移 `pan`：

```text
screen = pan + world * zoom
```

示意圖：

```text
以畫面中心為原點

0 -------- pan ----------- + -------- world * zoom ----------- = screen
```

也可以想成：

```text
最後畫面位置 = 縮放後的位置 + 整體平移量
```

---

## 這個公式到底在解決什麼？

我們真正想知道的是：

> 滑鼠現在指到的畫面位置，對應到內容中的哪個點？

也就是說，我們已知的是：

* `screen`：滑鼠目前所在的畫面位置
* `pan`：目前的平移量
* `zoom`：目前的縮放倍率

那就可以反推出：

* `world`：滑鼠底下對應的內容點

所以把公式移項：

```text
world = (screen - pan) / zoom
```

---

## 為什麼是先減 `pan`，再除 `zoom`？

這不是只是在做代數移項，
它其實對應兩個很直覺的還原步驟。

### 第一步：先減掉 `pan`

```text
screen - pan
```

意思是：

> 先把「整體平移」造成的影響拿掉

因為現在看到的 `screen`，已經包含整個內容被平移後的位置了。

---

### 第二步：再除掉 `zoom`

```text
(screen - pan) / zoom
```

意思是：

> 再把「縮放」造成的距離放大或縮小還原回去

這樣就能回到內容本來的座標，也就是 `world`。

---

## 用一句話理解這個反推公式

```text
先扣掉平移，再還原縮放
```

最後就能知道：

> 滑鼠底下對應的是哪個內容點

---

## 這和 `zoomAroundPoint()` 有什麼關係？

`zoomAroundPoint()` 真正先做的事情，就是：

> **根據滑鼠目前所在的畫面位置，先找出滑鼠底下對應的是哪個 `world` 點。**

因為只有先找到這個 `world` 點，
後面你才能在縮放之後重新計算 `pan`，讓：

> 縮放前後都是同一個 `world` 點留在滑鼠底下

---

# 7. 用一組固定數字一路算到底

現在我們用一組固定數字，真的把公式算一次。

假設目前：

```text
currentPan = 100
currentZoom = 1
anchorOffset = 300
```

這裡的意思分別是：

* `currentPan = 100`：整個內容目前往右平移了 100
* `currentZoom = 1`：目前沒有放大縮小
* `anchorOffset = 300`：滑鼠目前位於**相對畫面中心右邊 300** 的位置

而此時，滑鼠所在的畫面位置，就是我們公式裡的：

```text
screen = 300
```

---

## 縮放前：先反推出滑鼠底下是哪個內容點

套用公式：

```text
screen = pan + world * zoom
```

把目前的數字代進去：

```text
300 = 100 + world * 1
```

移項後得到：

```text
world = 200
```

---

## 這個算式在圖上代表什麼？

```text
以畫面中心為原點

0 -------- 100 ---------------------- 300 --------------------→
            |                          |
           pan                    anchor / P
```

這張圖的意思是：

* `pan = 100`
* 滑鼠目前在 `300`
* 內容點 `P` 此時剛好也顯示在 `300`
* 所以滑鼠正指著 `P`

而從公式來看：

```text
screen - pan = 300 - 100 = 200
```

因為目前：

```text
zoom = 1
```

所以這個 `200` 不需要再縮放還原，直接就是：

```text
world = 200
```

---

## 所以我們得到什麼結論？

我們成功反推出：

> 滑鼠現在指到的內容點，是 `world = 200`

這個結論非常重要。

因為等等縮放之後，我們真正要固定的，不是單純的畫面位置 `300`，而是：

> **同一個內容點 `world = 200` 仍然要留在滑鼠底下**

---

# 8. 如果只改 zoom，不改 pan，會發生什麼事？

現在延續上一節算出的結果：

```text
currentPan = 100
world = 200
````

並假設接下來要把縮放倍率改成：

```text
nextZoom = 2
```

---

## 如果 `pan` 完全不變

也就是仍然維持：

```text
pan = 100
```

那同一個內容點 `world = 200`，在畫面上的位置就會變成：

```text
screen = pan + world * zoom
       = 100 + 200 * 2
       = 500
```

這裡的 `screen`，仍然是：

> **相對畫面中心的畫面位置**

---

## 這個算式代表什麼？

原本：

* `world = 200`
* `zoom = 1`

所以它顯示在：

```text
100 + 200 * 1 = 300
```

也就是滑鼠原本指到的地方。

但現在只把 `zoom` 改成 `2`，而 `pan` 不變：

* `world = 200`
* `zoom = 2`
* `pan = 100`

所以它就會跑到：

```text
100 + 200 * 2 = 500
```

也就是說：

> 同一個內容點，因為放大後離中心更遠了，所以被推到了 `500`

---

## 圖上怎麼看？

```text
以畫面中心為原點

0 -------- 100 ---------------------- 300 ------------ 500 ----→
            |                          |               |
           pan                        滑鼠          P（跑掉後）
```

這張圖表示：

* 滑鼠還停在 `300`
* 但原本滑鼠底下的內容點 `P`
* 因為只做了 `zoom`，現在跑到了 `500`

---

## 這代表什麼？

這就表示：

> **滑鼠底下已經不是原本那個內容點了**

也就是：

> 縮放前後，滑鼠底下不再是同一個 `world = 200`

這就叫做：

```text
沒有釘住
```

---

# 9. 那新的 pan 應該是多少？

現在我們已經知道兩件事：

- 縮放前，滑鼠底下對應的內容點是：

```text
world = 200
````

* 如果只把 `zoom` 改成 `2`，但不改 `pan`，那這個點會跑到：

```text
screen = 500
```

但這不是我們想要的。

---

## 我們真正想要的是什麼？

我們真正想要的是：

> **縮放後，`world = 200` 這個點，仍然要出現在滑鼠所在的畫面位置**

而滑鼠目前所在的位置是：

```text
screen = 300
```

也就是說，我們要強制滿足：

> 同一個內容點，縮放前後都還留在 `screen = 300`

這才叫做「釘住」。

---

## 把條件直接代進核心公式

我們的核心公式是：

```text
screen = pan + world * zoom
```

現在把目標條件代進去：

* `screen = 300`
* `world = 200`
* `zoom = 2`
* `pan` 改成未知數 `nextPan`

就得到：

```text
300 = nextPan + 200 * 2
```

也就是：

```text
300 = nextPan + 400
```

解出來：

```text
nextPan = -100
```

---

## `nextPan = -100` 代表什麼？

這個負號很重要。

```text
nextPan = -100
```

表示的是：

> **整個內容要往左平移 100**

不是只移動某一個點，
而是把整體內容一起往左拉。

這樣原本因為放大而跑到右邊的 `P`，才會被拉回來。

---

## 圖上怎麼看？

```text
以畫面中心為原點

0 ----(-100)-------------------------300-----------------------→
        |                             |
     nextPan                       anchor / P
```

這張圖表示：

* 新的平移量變成 `-100`
* 內容點 `P` 經過放大後，再配合新的 `pan`
* 最後又重新出現在滑鼠所在的 `300`

所以滑鼠又重新指回同一個內容點。

---

## 這一步的本質是什麼？

這一步不是在「硬把點拖回來」，
而是在做這件事：

> **反推出整體平移量應該是多少，才能讓指定的 world 點出現在指定的 screen 位置**

也就是：

* `world` 已經知道
* `zoom` 已經知道
* `screen` 也知道
* 所以反過來求 `nextPan`

---

## 這就叫做「把點釘住」

因為你固定的不是單純某個畫面像素，
而是這件事：

> **縮放前後，滑鼠底下都是同一個 `world = 200`**

這才是 `zoomAroundPoint()` 真正想做到的效果。

---

# 10. 一步一步推導出 `nextPan`

現在正式來推導 `nextPan` 是怎麼來的。

整個推導的核心目標只有一個：

> **縮放前後，滑鼠底下都要是同一個 `world` 點。**

所以我們會分成兩步：

```text
1. 先找出縮放前滑鼠底下是哪個 world 點
2. 再反推出縮放後需要多少 nextPan，才能讓它留在原位
```

---

## 第一步：先找出滑鼠底下是哪個 `world` 點

縮放前，滑鼠所在的畫面位置是 `anchorOffset`。

而在這份推導裡：

> `anchorOffset` 就是以畫面中心為原點時的 `screen`

所以可以直接套用核心公式：

```text
screen = pan + world * zoom
```

變成：

```text
anchorOffset = currentPan + world * currentZoom
```

---

### 這個式子在圖上代表什麼？

```text
以畫面中心為原點

0 -------- currentPan ------------------ anchorOffset --------→
          |<--------------- 差值 -------------------->|
```

圖上的意思是：

* `currentPan` 是目前整體平移量
* `anchorOffset` 是滑鼠所在的畫面位置
* 兩者的差：

```text
anchorOffset - currentPan
```

代表的是：

> 把平移拿掉之後，剩下「縮放後的內容距離」

而這個距離還包含了 `currentZoom` 的影響。

所以還要再除以 `currentZoom`，才能還原出原本的內容座標：

```text
world = (anchorOffset - currentPan) / currentZoom
```

---

## 第二步：縮放後，要求同一個 `world` 點還在 `anchorOffset`

現在我們已經知道：

> 滑鼠底下對應的是某個固定的 `world` 點

接下來要做的事是：

> 縮放之後，讓同一個 `world` 點仍然出現在同一個畫面位置 `anchorOffset`

所以再次套用核心公式：

```text
screen = pan + world * zoom
```

但這次改成縮放後的狀態：

* `screen = anchorOffset`
* `pan = nextPan`
* `zoom = nextZoom`

得到：

```text
anchorOffset = nextPan + world * nextZoom
```

移項後：

```text
nextPan = anchorOffset - world * nextZoom
```

---

## 把上一步求出的 `world` 代進來

上一步已經知道：

```text
world = (anchorOffset - currentPan) / currentZoom
```

代回去：

```text
nextPan = anchorOffset - ((anchorOffset - currentPan) / currentZoom) * nextZoom
```

這就是最原始、最直觀的版本。

它的意思其實很簡單：

```text
1. 先根據目前狀態，反推出滑鼠底下是哪個 world 點
2. 再根據新的 zoom，反推出要多少 nextPan，才能讓它仍然留在 anchorOffset
```

---

## 這一節最重要的理解方式

這個推導不是在硬背公式，
而是在做兩次反推：

```text
第一次反推：由畫面位置反推出 world
第二次反推：由目標位置反推出 nextPan
```

所以 `zoomAroundPoint()` 的本質就是：

> **先認出滑鼠底下是哪個內容點，再算出新的平移量把它留住。**

---

# 11. 為什麼可以整理成程式碼裡那個寫法？

前一節我們得到的，是最原始、最直觀的推導式：

```text
nextPan = anchorOffset - ((anchorOffset - currentPan) / currentZoom) * nextZoom
````

這個版本很適合理解「它是怎麼推導出來的」。

但在程式碼裡，通常會把它整理成更短、比較好實作的形式：

```ts
const zoomRatio = nextZoom / currentZoom

nextPan = currentPan + (anchorOffset - currentPan) * (1 - zoomRatio)
```

這不是另一套公式，
而是**同一件事的代數整理版**。

---

## 第一步：先把縮放倍率合併成 `zoomRatio`

先定義：

```ts
const zoomRatio = nextZoom / currentZoom
```

那原本的式子就可以寫成：

```text
nextPan
= anchorOffset - ((anchorOffset - currentPan) / currentZoom) * nextZoom
= anchorOffset - (anchorOffset - currentPan) * (nextZoom / currentZoom)
= anchorOffset - (anchorOffset - currentPan) * zoomRatio
```

到這裡為止，只是把：

```text
nextZoom / currentZoom
```

改寫成：

```text
zoomRatio
```

讓式子更短一點。

---

## 第二步：把 `anchorOffset` 改寫一下

接下來這一步看起來最像魔法，其實只是刻意改寫：

```text
anchorOffset = currentPan + (anchorOffset - currentPan)
```

為什麼要這樣寫？

因為我們想把式子裡共同出現的：

```text
(anchorOffset - currentPan)
```

提出來。

所以：

```text
nextPan
= anchorOffset - (anchorOffset - currentPan) * zoomRatio
```

可以改寫成：

```text
nextPan
= currentPan + (anchorOffset - currentPan)
  - (anchorOffset - currentPan) * zoomRatio
```

---

## 第三步：把共同項提出來

現在後兩項都有共同的：

```text
(anchorOffset - currentPan)
```

所以可以提出來：

```text
nextPan
= currentPan + (anchorOffset - currentPan) * (1 - zoomRatio)
```

這就變成程式碼裡的形式了。

---

## 這個寫法為什麼比較適合放在程式碼裡？

因為它可以直接看成：

```text
nextPan = currentPan + 補償量
```

其中補償量是：

```text
(anchorOffset - currentPan) * (1 - zoomRatio)
```

也就是：

> 根據「滑鼠和目前平移基準之間的距離」
> 再乘上「這次縮放造成的比例變化」
> 算出應該補多少平移量

所以這個版本雖然不像原始式子那麼直觀地顯示推導過程，
但它更短，也更像實際會寫進函式裡的形式。

---

## 這一節最重要的結論

```text
原始公式負責說明「怎麼推導」
整理後的公式負責方便「怎麼實作」
```

兩者不是不同邏輯，
只是**同一個公式的兩種表達方式**。

---

# 12. 用圖理解 `(anchorOffset - currentPan)`

這段最容易讓人覺得抽象：

```ts
(anchorOffset - currentPan)
```

但它其實不是什麼神秘值，
它表示的是：

> **把整體平移 `currentPan` 扣掉之後，滑鼠位置還剩下多少距離**

也可以理解成：

> **滑鼠位置相對於內容目前平移後起點，還多出多少**

---

## 圖上怎麼看？

```text
以畫面中心為原點

0 -------- 100 ---------------------- 300 --------------------→
          currentPan              anchorOffset
```

從 `currentPan = 100` 到 `anchorOffset = 300`，中間這段距離就是：

```text
anchorOffset - currentPan = 200
```

如果畫成更明確一點：

```text
0 -------- 100 ---------------------------- 300 --------------------→
          |<-------- 剩下這段距離 = 200 -------->|
       currentPan                           anchorOffset
```

這個 `200` 的意思是：

> **滑鼠位置扣掉平移之後，剩下的那段距離**

---

## 這段距離為什麼重要？

因為根據核心公式：

```text
anchorOffset = currentPan + world * currentZoom
```

把 `currentPan` 移到左邊後：

```text
anchorOffset - currentPan = world * currentZoom
```

所以：

```text
(anchorOffset - currentPan)
```

其實代表的不是原本的 `world`，
而是：

> **`world` 經過目前縮放後，在畫面上呈現出來的距離**

---

## 為什麼 `zoom = 1` 時，它剛好等於 `world`？

如果：

```text
currentZoom = 1
```

那就表示目前沒有放大縮小。

所以：

```text
world * currentZoom = world * 1 = world
```

也就是說：

```text
anchorOffset - currentPan = world
```

這就是為什麼你前面那個例子裡：

```text
300 - 100 = 200
```

可以直接得到：

```text
world = 200
```

---

## 如果 `zoom != 1` 呢？

那這個差值就不再是原本的 `world`，
而是：

```text
world * currentZoom
```

例如：

* `world = 200`
* `currentZoom = 2`

那麼：

```text
anchorOffset - currentPan = 400
```

圖上可以想成：

```text
原本內容距離：        200
縮放 2 倍後畫面距離：  400
```

所以這個差值其實是在告訴你：

> **滑鼠底下那個內容點，經過目前縮放後，在畫面上離 `currentPan` 有多遠**

---

# 13. 用圖理解 `(1 - zoomRatio)`

在這個公式裡：

```ts
nextPan = currentPan + (anchorOffset - currentPan) * (1 - zoomRatio)
````

最容易抽象的除了：

```ts
(anchorOffset - currentPan)
```

另一個就是：

```ts
(1 - zoomRatio)
```

它真正控制的是：

> **補償量要往哪個方向加，以及要補多少**

也就是說：

* `(anchorOffset - currentPan)` 提供一個「目前距離基底」
* `(1 - zoomRatio)` 決定這個基底要乘上什麼比例，形成最後的補償量

---

## 先記住 `zoomRatio` 是什麼

```ts
zoomRatio = nextZoom / currentZoom
```

所以：

* `zoomRatio > 1`：代表放大
* `zoomRatio < 1`：代表縮小
* `zoomRatio = 1`：代表倍率沒變，不需要補償

而：

```text
1 - zoomRatio
```

就會直接反映這次縮放到底是放大還是縮小。

---

## 情況 A：放大

如果：

```text
currentZoom = 1
nextZoom = 2
zoomRatio = 2
1 - zoomRatio = -1
```

這時 `(1 - zoomRatio)` 是負數。

負數代表的意思是：

> **補償量要往反方向加**

為什麼？

因為內容放大後，會離中心更遠。
如果不補償，滑鼠底下原本那個點就會被推得更遠。

所以這時必須把整個內容往反方向拉，才能把它補回來。

圖：

```text
放大前

0 ---------------- P ----------------------------→
                   ↑
                  滑鼠

放大後（如果不補 pan）

0 -------------------------------- P ----------→
                   ↑
                  滑鼠
               （P 被推遠）
```

```text
所以要補反方向的 pan

←-------------- 把內容拉回來 -------------------

```

這就是為什麼放大時：

```text
1 - zoomRatio < 0
```

---

## 情況 B：縮小

如果：

```text
currentZoom = 2
nextZoom = 1
zoomRatio = 0.5
1 - zoomRatio = 0.5
```

這時 `(1 - zoomRatio)` 是正數。

正數代表的意思是：

> **補償量會沿著原本距離的方向去補**

為什麼？

因為內容縮小後，原本離中心較遠的點，會自動往中心縮回來。

如果不補償，滑鼠底下原本那個點就會往內縮，離開滑鼠位置。

所以這時就要再補一段同方向的平移，把它推回滑鼠底下。

圖：

```text
縮小前

0 ----------------------------- P --------------------------→
                                ↑
                               滑鼠
```

```text
縮小後（如果不補 pan）

0 ---------------- P --------------------------------------→
                   ↑
               P 往中心縮回來
```

```text
所以要再補 pan，把它推回滑鼠底下
```

---

## 用一句話記住正負號的意義

```text
放大：1 - zoomRatio < 0，補償量反向
縮小：1 - zoomRatio > 0，補償量同向
```

---

## 那「補多少」又是誰決定的？

除了方向之外，`(1 - zoomRatio)` 的數值大小，也決定補償量的強弱。

例如：

* 從 `1 → 2`，`1 - zoomRatio = -1`
* 從 `1 → 1.1`，`1 - zoomRatio = -0.1`

這表示：

> 放大越多，補償量越大
> 縮放變化越小，補償量也越小

---

## 這一節最重要的結論

```text
(1 - zoomRatio)
```

不是神秘常數，
它只是用來描述：

> **這次縮放是放大還是縮小，以及需要補多少反向 / 同向的平移量**

# 14. 再看一次完整程式碼

```ts
export function zoomAroundPoint(
  currentPan: PanPoint,
  currentZoom: number,
  nextZoom: number,
  anchor: CanvasPoint,
  stageSize: StageSize,
) {
  const stageCenterX = stageSize.width / 2
  const stageCenterY = stageSize.height / 2
  const anchorOffsetX = anchor.x - stageCenterX
  const anchorOffsetY = anchor.y - stageCenterY
  const zoomRatio = nextZoom / (currentZoom || 1)

  return {
    x: currentPan.x + (anchorOffsetX - currentPan.x) * (1 - zoomRatio),
    y: currentPan.y + (anchorOffsetY - currentPan.y) * (1 - zoomRatio),
  }
}
```

# 15. 每一行到底在幹嘛？

前面我們已經把整個原理推完了。  
現在回到原始程式碼，直接看：

> **每一段程式碼在整個縮放流程裡扮演什麼角色**

---

## 1. 找出畫面中心

```ts
const stageCenterX = stageSize.width / 2
const stageCenterY = stageSize.height / 2
```

目的：

> 建立「以畫面中心為原點」的座標基準。

因為後面不管是：

* 滑鼠位置
* `pan`
* `world * zoom`

都要放到同一套「中心座標系」裡看，
所以第一步一定要先知道中心在哪裡。

圖：

```text
Stage（左上角為原點）

+--------------------------------------------------+
|                                                  |
|                                                  |
|                     ● center                     |
|                                                  |
|                                                  |
+--------------------------------------------------+
```

---

## 2. 把滑鼠從左上角座標，轉成相對中心座標

```ts
const anchorOffsetX = anchor.x - stageCenterX
const anchorOffsetY = anchor.y - stageCenterY
```

目的：

> 把滑鼠從「左上角原點座標」
> 改寫成
> 「相對畫面中心偏移多少」

也就是：

```text
anchorOffset = anchor - center
```

圖：

```text
左上角原點座標

+--------------------------------------------------+
|                                                  |
|                     ● center                     |
|                           \                      |
|                            \                     |
|                             ● anchor             |
|                                                  |
+--------------------------------------------------+

轉成中心座標後：

anchorOffset = anchor - center
```

所以這一步不是在算神秘公式，
而是在做一件事：

> **統一座標系**

---

## 3. 算出這次縮放是原本的幾倍

```ts
const zoomRatio = nextZoom / (currentZoom || 1)
```

目的：

> 用一個比值描述「這次縮放變化了多少」

例如：

* `1 -> 2`，`zoomRatio = 2`
* `2 -> 1`，`zoomRatio = 0.5`

這樣後面就能用同一套公式，同時處理：

* 放大
* 縮小

### 為什麼要寫 `(currentZoom || 1)`？

這通常是**防呆寫法**。

意思是：

> 如果 `currentZoom` 意外是 `0`、`undefined` 或其他 falsy 值，至少先用 `1`，避免除法出錯。

正常情況下：

* `currentZoom` 應該是有效縮放值
* 通常不應該是 `0`

所以這不是公式本質的一部分，
而比較像是實作上的安全保護。

---

## 4. 套用公式，算出新的 `pan`

```ts
x: currentPan.x + (anchorOffsetX - currentPan.x) * (1 - zoomRatio)
y: currentPan.y + (anchorOffsetY - currentPan.y) * (1 - zoomRatio)
```

這裡其實就是把前面推導完的結果，直接套到 X 和 Y 兩個方向。

如果先只看 X：

```ts
currentPan.x + (anchorOffsetX - currentPan.x) * (1 - zoomRatio)
```

可以拆成三個部分理解。

---

### (a) `currentPan.x`

這是：

> **目前的平移量**

也就是「現在整個內容已經被平移到哪裡」。

---

### (b) `(anchorOffsetX - currentPan.x)`

這是：

> **滑鼠位置扣掉目前平移後，剩下的那段距離**

圖：

```text
0 -------- currentPan.x ----------------- anchorOffsetX --------→
          |<---------------- 差值 --------------------->|
```

這段距離其實對應的是：

```text
world * currentZoom
```

如果 `currentZoom = 1`，它就剛好等於 `world`。

---

### (c) `(1 - zoomRatio)`

這是：

> **補償量的方向和比例**

* 放大時，`zoomRatio > 1`，所以 `(1 - zoomRatio)` 會是負數
  → 補償量往反方向，才能把被放大的內容拉回來

* 縮小時，`zoomRatio < 1`，所以 `(1 - zoomRatio)` 會是正數
  → 補償量沿著另一側補回去，維持滑鼠底下還是同一個點

---

## 所以整行的意思其實就是：

```text
新的 pan
= 舊的 pan
+ 根據滑鼠位置與目前內容距離算出的補償量
```

它本質上仍然是在做同一件事：

> **讓縮放前滑鼠底下的那個 world 點，縮放後仍然停在滑鼠底下**

只是前面推導用的是比較直觀的版本，
而這裡已經整理成比較適合直接寫進程式碼的形式。

---

## 最後用一句話總結整段程式碼

```text
1. 先把滑鼠轉成中心座標
2. 再算出這次縮放變了多少
3. 最後補上一個新的 pan，讓同一個內容點被釘在滑鼠底下
```

# 16. 用一張總圖把整個流程看完

前面講了很多段，現在用一組總圖把整個流程一次看完。

整件事其實只是在做一件事：

> **縮放前後，都讓滑鼠底下維持同一個 `world` 點**

---

## 縮放前

此時滑鼠所在的畫面位置是：

```text
anchorOffset
```

而滑鼠底下對應的內容點是某個固定的 `world` 點，所以它滿足：

```text
anchorOffset = currentPan + world * currentZoom
```

圖：

```text
以畫面中心為原點

0 -------- currentPan ---------------- anchorOffset ----------→
          |<--------- world * currentZoom --------->|
                              ↑
                            滑鼠 / P
```

這張圖表示：

* `currentPan` 是目前整體平移量
* `world * currentZoom` 是內容點經過目前縮放後的距離
* 兩者加起來，剛好落在滑鼠所在的 `anchorOffset`
* 所以滑鼠現在正指著內容點 `P`

---

## 縮放時，如果只改 `zoom`

如果這時候：

* `anchorOffset` 不變
  （因為滑鼠在畫面上的位置沒動）

* `world` 不變
  （因為我們要固定的是同一個內容點）

* `zoom` 改變

那麼：

```text
world * zoom
```

這段距離就會改變。

如果 `pan` 仍然維持 `currentPan` 不變，
那最後這個點在畫面上的位置就一定會改變。

圖：

```text
以畫面中心為原點

0 -------- currentPan ------------------- 新位置 ------------→
                                          ↑
                                       P 跑掉了
```

也就是說：

> 同一個 `world` 點，因為縮放改變，已經不再待在滑鼠底下。

---

## 所以要重新算 `nextPan`

接下來我們不改 `world`，
而是改：

```text
nextPan
```

讓縮放後同一個內容點重新滿足：

```text
anchorOffset = nextPan + world * nextZoom
```

圖：

```text
以畫面中心為原點

0 ---- nextPan ------------------------ anchorOffset --------→
      |<-------- world * nextZoom ------->|
                                      ↑
                                   滑鼠 / P
```

這張圖表示：

* `world` 還是同一個 `world`
* `nextZoom` 已經是新的縮放倍率
* 但因為重新算了 `nextPan`
* 所以這個點又被拉回到滑鼠所在的 `anchorOffset`

---

## 用一句話把整個流程記住

```text
先找出滑鼠底下是哪個 world 點
再根據新的 zoom，反推出新的 nextPan
```

這樣就能做到：

> **縮放前後，滑鼠底下始終是同一個內容點**

# 17. 最精簡的理解方式

如果你之後忘了公式，只要記住下面這段就夠了：

```text
縮放時，真正要固定的不是滑鼠這個座標本身，
而是滑鼠底下對應的那個內容點。

所以流程一定是：
1. 先反推出滑鼠底下是哪個 world 點
2. 改變 zoom
3. 再重算 pan，讓同一個 world 點回到原本的畫面位置
```

如果你想再記得更短一點，也可以直接記成：

```text
先找出滑鼠底下是哪個點
再把那個點釘回去
```

# 18. 一句話總結

> `zoomAroundPoint()` 不是單純地一邊縮放一邊平移，而是先認出滑鼠底下是哪個內容點，再用新的縮放倍率反推出 `pan`，把同一個點精準地釘回原位。

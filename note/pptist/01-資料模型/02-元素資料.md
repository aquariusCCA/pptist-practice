# 元素資料

這一篇要解決的核心問題是：

- `PPTElement` 到底是什麼？
- 為什麼它不是一個固定結構，而是一組 union type？
- 各種元素有哪些共用欄位，又有哪些專屬欄位？

對應原始碼：

- `PPTist-SourceCode/src/types/slides.ts`
- `PPTist-SourceCode/src/views/components/element`
- `PPTist-SourceCode/src/configs/element.ts`
- `PPTist-SourceCode/src/configs/shapes.ts`
- `PPTist-SourceCode/src/configs/lines.ts`
- `PPTist-SourceCode/src/configs/chart.ts`
- `PPTist-SourceCode/src/configs/latex.ts`
- `PPTist-SourceCode/src/configs/imageClip.ts`
- `PPTist-SourceCode/src/utils/element.ts`

---

## 先一句話理解

`PPTElement` 是投影片上所有元素的**總型別**。

但它不是一個「把所有欄位全部塞在一起的大物件」，而是：

- 先有一組多數元素共用的 base 欄位
- 再依照 `type` 分成不同子型別

所以理解 `PPTElement` 的正確方式是：

> 先看共用欄位，再看每個 `type` 對應的專屬資料。

---

## 先看共用枚舉

`slides.ts` 另外還定義了一個 `ElementTypes`，它和 `PPTElement` 的子型別一一對應：

```ts
TEXT = 'text'
IMAGE = 'image'
SHAPE = 'shape'
LINE = 'line'
CHART = 'chart'
TABLE = 'table'
LATEX = 'latex'
VIDEO = 'video'
AUDIO = 'audio'
```

這個 enum 的用途是讓程式在不同地方可以用同一組元素類型字串，不必自己重打字串常數。

---

## `PPTElement` 的整體結構

可以先用這張簡化圖建立印象：

```text
PPTElement
├─ base 欄位
└─ 子型別
   ├─ text
   ├─ image
   ├─ shape
   ├─ line
   ├─ chart
   ├─ table
   ├─ latex
   ├─ video
   └─ audio
```

這表示你在看某個元素時，通常要分兩步：

1. 先看它是不是所有元素都共用的欄位
2. 再看它的 `type` 是什麼，決定要對應哪個子型別

---

## 共用基底欄位

`PPTTextElement`、`PPTImageElement`、`PPTShapeElement`、`PPTChartElement`、`PPTTableElement`、`PPTLatexElement`、`PPTVideoElement`、`PPTAudioElement` 都繼承 `PPTBaseElement`。

`PPTBaseElement` 的欄位是：

- `id`
- `left`
- `top`
- `width`
- `height`
- `rotate`
- `lock?`
- `groupId?`
- `link?`
- `name?`

這些欄位主要描述的是：

- 元素是誰 → `id`
- 放在哪裡 → `left`、`top`
- 大小多大 → `width`、`height`
- 旋轉多少 → `rotate`
- 是否被鎖定 → `lock?`
- 是否屬於群組 → `groupId?`
- 是否帶有連結 → `link?`
- 是否有自訂名稱 → `name?`

也就是說，base 欄位負責描述元素的**通用幾何位置與共同行為資訊**。

---

## 基底欄位補充

### `link?`
- 型別是 `PPTElementLink`
- 結構為：

```ts
{ type: 'web' | 'slide', target: string }
```

這表示元素連結可能指向：

- 外部網址
- 另一張投影片

### `groupId?`
- 表示這個元素屬於哪個群組
- 用於群組操作或群組選取

### `lock?`
- 表示元素是否鎖定
- 鎖定後通常不允許直接被拖曳或修改

### `outline?`、`shadow?`
- 這兩個不是 base 欄位，而是常出現在文字、圖片、圖形、圖表與線條等型別中的視覺設定
- 對應型別分別是 `PPTElementOutline` 與 `PPTElementShadow`

### `ImageOrShapeFlip`
- 圖片與圖形共用的翻轉設定
- 欄位只有：
  - `flipH?`
  - `flipV?`

### `Gradient`
- 漸層資料結構
- 欄位包含：
  - `type`
  - `colors`
  - `rotate`

---

## `PPTElement` 的子型別

`PPTElement` 是以下子型別的 union：

- `text`
- `image`
- `shape`
- `line`
- `chart`
- `table`
- `latex`
- `video`
- `audio`

這代表：

- 並不是每個元素都有同樣的欄位
- `type` 會決定這個元素實際擁有哪一組專屬欄位

---

## 文字元素

### `PPTTextElement`

常見欄位：

- `type: 'text'`
- `content`
- `defaultFontName`
- `defaultColor`
- `outline?`
- `fill?`
- `lineHeight?`
- `wordSpace?`
- `opacity?`
- `shadow?`
- `paragraphSpace?`
- `vertical?`
- `textType?`

一句話理解：

> 文字元素除了內容本身，還會保存預設字型、顏色、文字排版與視覺效果。

### `TextType`

`textType` 的值不是任意字串，而是具語意分類，例如：

- `title`
- `subtitle`
- `content`
- `item`
- `itemTitle`
- `notes`
- `header`
- `footer`
- `partNumber`
- `itemNumber`

這表示文字元素除了顯示內容，也可能帶有版型語意。

---

## 圖片元素

### `PPTImageElement`

常見欄位：

- `type: 'image'`
- `fixedRatio`
- `src`
- `outline?`
- `filters?`
- `clip?`
- `flipH?`
- `flipV?`
- `shadow?`
- `radius?`
- `colorMask?`
- `imageType?`

一句話理解：

> 圖片元素除了圖片來源，也會保存裁切、翻轉、濾鏡、圓角與顏色遮罩等視覺處理資訊。

### `ImageElementFilters`
- 對應圖片濾鏡設定
- 欄位使用 CSS filter 名稱作為 key
- 實際可用的 key 包含：
  - `blur`
  - `brightness`
  - `contrast`
  - `grayscale`
  - `saturate`
  - `hue-rotate`
  - `sepia`
  - `invert`
  - `opacity`

### `ImageElementFilterKeys`
- 這是圖片濾鏡的 key union
- 和 `ImageElementFilters` 對應，但前者是 key 集合，後者是物件結構

### `ImageType`

`imageType` 的值有：

- `pageFigure`
- `itemFigure`
- `background`

這表示圖片也可能有不同的語意用途，而不只是單純顯示一張圖。

### 其他圖片相關型別

- `filters` 對應 `ImageElementFilters`
- `clip` 對應 `ImageElementClip`
- `clip.range` 的型別是 `[[number, number], [number, number]]`
- `clip.shape` 對應 `configs/imageClip.ts` 裡的裁切形狀設定

---

## 圖形元素

### `PPTShapeElement`

常見欄位：

- `type: 'shape'`
- `viewBox`
- `path`
- `fixedRatio`
- `fill`
- `gradient?`
- `pattern?`
- `outline?`
- `opacity?`
- `flipH?`
- `flipV?`
- `shadow?`
- `special?`
- `text?`
- `pathFormula?`
- `keypoints?`

一句話理解：

> 圖形元素本質上是可被描繪的向量形狀，並可附帶填色、漸層、圖樣與內嵌文字。

### `ShapeText`

如果圖形內有文字，`text` 的結構通常包含：

- `content`
- `defaultFontName`
- `defaultColor`
- `align`
- `type?`

`align` 的值有：

- `top`
- `middle`
- `bottom`

這表示圖形除了形狀本身，也可能承載一段內部文字。

### `ShapePathFormulasKeys`
- 這是圖形路徑公式的 enum
- 對應一些需要在縮放時重新計算 path 的特殊圖形
- 值會對應到 `configs/shapes.ts` 內的形狀定義
- `special?` 則是用來標記難以正常解析的形狀

---

## 線條元素

### `PPTLineElement`

這個型別和一般 base element 不完全一樣，它比較接近：

- `Omit<PPTBaseElement, 'height' | 'rotate'>`

常見欄位：

- `type: 'line'`
- `start`
- `end`
- `style`
- `color`
- `points`
- `shadow?`
- `broken?`
- `broken2?`
- `curve?`
- `cubic?`

### 這裡最容易誤解的地方

線條元素和一般矩形元素不同，所以有兩個重要特點：

- 線條沒有 `height`
- 線條沒有 `rotate`

這是因為線條的幾何描述方式與一般盒模型元素不同，它比較依賴起點、終點與折線控制資訊。

### 補充

- `points` 是 `[LinePoint, LinePoint]`
- `LinePoint` 的值有：
  - `''`
  - `arrow`
  - `dot`

這表示線條的頭尾可以有不同端點樣式。

## 最小例子：不同 `type` 會長得不一樣

### 文字元素例子

```ts
const textElement = {
  id: 'el-1',
  type: 'text',
  left: 100,
  top: 80,
  width: 300,
  height: 60,
  rotate: 0,
  content: '營收成長分析',
  defaultFontName: 'Microsoft YaHei',
  defaultColor: '#333333'
}
```

### 圖片元素例子

```ts
const imageElement = {
  id: 'el-2',
  type: 'image',
  left: 80,
  top: 160,
  width: 240,
  height: 160,
  rotate: 0,
  fixedRatio: true,
  src: '/images/chart.png'
}
```

### 線條元素例子

```ts
const lineElement = {
  id: 'el-3',
  type: 'line',
  left: 100,
  top: 100,
  width: 300,
  start: [0, 0],
  end: [300, 0],
  style: 'solid',
  color: '#333333',
  points: ['', 'arrow']
}
```

這三個例子最想表達的是：

- 它們都屬於 `PPTElement`
- 但實際欄位會依 `type` 而不同
- 所以不能把 `PPTElement` 當成單一固定結構來看

---

## 怎麼讀這一類型別最有效

看 `PPTElement` 時，建議固定用下面順序：

1. 先看 base 欄位
2. 再看 `type`
3. 再看該子型別的專屬欄位
4. 最後看巢狀型別與 union 值

例如：

- 看圖片時，不只看 `src`，還要看 `filters`、`clip`、`imageType`
- 看圖形時，不只看 `path`，還要看 `fill`、`gradient`、`text`
- 看線條時，要特別注意它不是一般矩形元素

---

## 小結

如果只看外層筆記，很容易把 `PPTElement` 誤解成「很多欄位拼在一起的大型別」。

更準確的理解方式是：

- 先看共用 base 欄位
- 再看每個 `type` 對應的子型別
- 再看各自的巢狀型別與 union 值
- 特別留意少數特殊型別，例如 `line`

只要用這個方式閱讀，你會更接近 `pptist` 原始碼的實際設計，也更容易對照後續的元件實作。

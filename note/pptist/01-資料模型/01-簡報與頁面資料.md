# 簡報與頁面資料

這一篇對照 `slides store` 與 `Slide` 的資料結構。

對應原始碼：

- `PPTist-SourceCode/src/types/slides.ts`
- `PPTist-SourceCode/src/store/slides.ts`

## `SlidesState`

`slides store` 裡主要有這些欄位：

- `title`
- `theme`
- `slides`
- `slideIndex`
- `viewportSize`
- `viewportRatio`
- `templates`

### 欄位意義

- `title`
  - 簡報標題
- `theme`
  - 簡報主題樣式
- `slides`
  - 所有投影片資料
- `slideIndex`
  - 目前正在編輯的頁面索引
- `viewportSize`
  - 視窗基準寬度，用來計算畫布縮放
- `viewportRatio`
  - 視窗比例，預設是 `0.5625`，也就是 `16:9`
- `templates`
  - 範本清單

### 原始碼中的預設值

在 `src/store/slides.ts`，預設值大致是：

- `title` = `未命名演示文稿`
- `slides` = 空陣列
- `slideIndex` = `0`
- `viewportSize` = `1000`
- `viewportRatio` = `0.5625`
- `templates` = 4 個內建範本

`theme` 也有預設結構，包含：

- `themeColors`
- `fontColor`
- `fontName`
- `backgroundColor`
- `shadow`
- `outline`

## `Slide`

`Slide` 是單頁投影片的資料結構。

### 欄位

- `id`
- `elements`
- `notes?`
- `remark?`
- `background?`
- `animations?`
- `turningMode?`
- `sectionTag?`
- `type?`

### 欄位說明

- `id`
  - 投影片 ID
- `elements`
  - 這一頁上的所有元素
- `notes?`
  - 投影片註解，型別是 `Note[]`
- `remark?`
  - 頁面備註文字
- `background?`
  - 投影片背景，型別是 `SlideBackground`
- `animations?`
  - 元素動畫，型別是 `PPTAnimation[]`
- `turningMode?`
  - 投影片轉場模式
- `sectionTag?`
  - 分段標記，型別是 `SectionTag`
- `type?`
  - 投影片類型

## 相關巢狀型別

### `Note`

`notes` 不是單純字串陣列，而是結構化註解。

`Note` 的欄位：

- `id`
- `content`
- `time`
- `user`
- `elId?`
- `replies?`

`replies` 的型別是 `NoteReply[]`。

### `SectionTag`

`sectionTag` 不是字串，而是：

- `id`
- `title?`

這表示它不只是標籤名稱，還有自己的識別資訊。

### `SlideBackground`

背景型別是：

- `type`
- `color?`
- `image?`
- `gradient?`

其中 `type` 的 union 值是：

- `solid`
- `image`
- `gradient`

### `PPTAnimation`

動畫欄位主要是：

- `id`
- `elId`
- `effect`
- `type`
- `duration`
- `trigger`

`trigger` 的值有：

- `click`
- `meantime`
- `auto`

### `SlideType`

投影片類型的 union 值是：

- `cover`
- `contents`
- `transition`
- `content`
- `end`

## 這一頁與 store 的關係

`slides store` 不只保存 `slides` 陣列，還會提供：

- `currentSlide`
- `currentSlideAnimations`
- `formatedAnimations`

也就是說，`Slide` 是資料本體，而 `store` 會再根據它做出目前頁面與動畫的衍生狀態。

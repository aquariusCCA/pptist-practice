# 資料模型總覽

`pptist` 的資料模型，建議不要一開始就把所有型別一起看。
比較好的方式是先抓住主線，再逐層往下拆。

這一節的主線是：

1. 整份簡報在編輯器裡怎麼被保存
2. 單頁投影片怎麼被表示
3. 頁面上的元素怎麼被表示
4. 背景、動畫、註解、分段標記怎麼補充頁面內容

對照原始碼主要看：

- `src/types/slides.ts`
- `src/store/slides.ts`

---

## 一句話先理解整體

可以先把 `pptist` 的資料模型想成三層：

- **第一層：整份簡報** → `SlidesState`
- **第二層：單頁投影片** → `Slide`
- **第三層：頁面元素** → `PPTElement`

而動畫、背景、註解、分段標記等資料，則是附著在 `Slide` 上的補充內容。

也就是說，整體結構不是「一個超大物件包全部」，而是：

```text
SlidesState
├─ theme
└─ slides: Slide[]
   └─ elements: PPTElement[]
```

這是理解整個資料模型最重要的骨架。

---

## 為什麼要先分層看？

因為如果直接看原始碼中的型別清單，很容易出現兩個問題：

1. 會把很多型別看成彼此獨立，卻不知道它們實際掛在哪裡
2. 會把內容資料與編輯器狀態資料混在一起

所以在看 `pptist` 的資料模型時，建議先問自己三個問題：

- 這是整份簡報的資料，還是單頁的資料？
- 這是頁面元素本體，還是元素的補充設定？
- 這是會被保存的內容，還是 store 為了編輯方便維護的狀態？

只要先分清楚這三件事，後面看型別就會順很多。

---

## 核心資料區塊

### 1. 簡報層級：`SlidesState`

這是 `slides store` 的主要 state。
它不是單純的內容資料，而是「**整份簡報在編輯器中的狀態容器**」。

它會包含：

- 簡報標題
- 簡報主題
- 所有投影片資料
- 目前所在頁面索引
- 視窗基準寬度與比例
- 範本清單

所以 `SlidesState` 比較像：

> 編輯器目前正在管理的整份簡報狀態。

---

### 2. 頁面層級：`Slide`

`Slide` 是單頁投影片的資料結構，也是簡報內容的核心單位。

一頁投影片通常會包含：

- 頁面 ID
- 頁面上的元素清單
- 註解 `notes?: Note[]`
- 文字備註 `remark?: string`
- 背景
- 動畫
- 分段標記
- 頁面類型
- 轉場模式

所以 `Slide` 可以理解成：

> 一頁投影片真正被保存下來的內容本體。

---

### 3. 元素層級：`PPTElement`

`PPTElement` 是頁面上所有元素的總型別。

它本身不是單一固定結構，而是一組 union type，底下包含：

- 文字
- 圖片
- 圖形
- 線條
- 圖表
- 表格
- LaTeX
- 影片
- 音訊

所以理解 `PPTElement` 的重點不是把所有欄位背起來，而是先知道：

- 哪些欄位是多數元素共用的 base 欄位
- 哪些欄位只屬於特定 `type`

---

### 4. 頁面補充資料

除了 `Slide` 與 `PPTElement`，頁面還會掛一些補充型別：

- `PPTAnimation`
- `SlideBackground`
- `Note`
- `SectionTag`

這些型別不是頁面的主幹，但它們會補完整個頁面的表現與語意。

例如：

- `SlideBackground` 決定頁面背景
- `PPTAnimation` 決定元素動畫
- `Note` 表示註解資料
- `SectionTag` 表示分段資訊

---

## 最小關係圖

把主要型別放在一起看，可以先記成下面這樣：

```text
SlidesState
├─ title
├─ theme
├─ slides: Slide[]
├─ slideIndex
├─ viewportSize
├─ viewportRatio
└─ templates

Slide
├─ id
├─ elements: PPTElement[]
├─ notes?: Note[]
├─ remark?: string
├─ background?: SlideBackground
├─ animations?: PPTAnimation[]
├─ turningMode?: TurningMode
├─ sectionTag?: SectionTag
└─ type?: SlideType

PPTElement
├─ base 欄位
└─ 子型別
   ├─ text
   ├─ image
   ├─ shape
   ├─ line
   ├─ chart
   ├─ table
   ├─ latex
   ├─ video
   └─ audio
```

## 最小例子：一份簡化後的內容長什麼樣

下面是一個很小的示意例子，目的是幫你建立形狀感：

```ts
const slidesState = {
  title: '未命名演示文稿',
  theme: {
    themeColors: ['#5b9bd5', '#ed7d31', '#a5a5a5', '#ffc000', '#4472c4', '#70ad47'],
    fontColor: '#333',
    fontName: '',
    backgroundColor: '#fff',
    shadow: {
      h: 3,
      v: 3,
      blur: 2,
      color: '#808080',
    },
    outline: {
      width: 2,
      color: '#525252',
      style: 'solid',
    },
  },
  slides: [
    {
      id: 'slide-1',
      elements: [
        {
          id: 'el-1',
          type: 'text',
          left: 100,
          top: 80,
          width: 300,
          height: 60,
          rotate: 0,
          content: 'Hello PPTist',
          defaultFontName: 'Microsoft YaHei',
          defaultColor: '#333333'
        }
      ],
      background: {
        type: 'solid',
        color: '#ffffff'
      },
      animations: [],
      notes: [],
      remark: ''
    }
  ],
  slideIndex: 0,
  viewportSize: 1000,
  viewportRatio: 0.5625,
  templates: []
}
```

這個例子不用追求完全等同原始碼，而是要看懂三件事：

- 整份簡報由 `SlidesState` 包起來
- `theme` 是 `SlidesState` 的一部分，不是單獨放在外面的設定
- 每頁內容放在 `slides` 裡
- 每頁的元素放在 `elements` 裡

## 本節重點

在 `pptist` 裡，最重要的不是記住一大堆型別名稱，而是先掌握下面這組骨架：

- `SlidesState`：整份簡報的編輯器狀態容器
- `Slide`：單頁投影片內容本體
- `PPTElement`：頁面元素總型別
- `PPTAnimation`、`SlideBackground`、`Note`、`SectionTag`：頁面補充資料

如果要理解資料結構，建議依序看：

1. `SlidesState` 怎麼承載整份簡報
2. `Slide` 怎麼描述一頁投影片
3. `PPTElement` 怎麼描述頁面上的元素
4. 頁面補充型別怎麼附著在 `Slide` 上

如果你想理解編輯器狀態、getter、快照或歷史紀錄，請移到 `05-store與狀態流`，不要和本節混在一起看。

# `wheel`：滾輪事件

`wheel` 是瀏覽器原生的**滾輪輸入事件**，屬於 `WheelEvent`，**不是 Pointer Events**。

它代表的是：

> **使用者發出了滾輪輸入。**

要注意，這不等於畫面已經發生捲動。
也就是說：

* `wheel`：輸入事件
* `scroll`：捲動結果事件

這個差別很重要。
在畫布系統中，我們通常不是想讓頁面真的捲動，而是想把這個滾輪輸入改解讀成：

* 畫布縮放
* 視角遠近調整
* 水平或垂直平移
* 攝影機控制

---

## 核心概念

`wheel` 的本質不是「頁面在滾」，而是：

> **使用者透過滑鼠滾輪、觸控板或其他滾動裝置，送出一段連續的滾動輸入。**

所以在畫布裡，`wheel` 最常見的用途不是做一般頁面捲動，而是把這個輸入重新解讀成畫布操作，尤其是**縮放**。

這也是為什麼在畫布、地圖、白板、簡報編輯器這類系統中，`wheel` 幾乎都會扮演重要角色。

---

## 重點

* `wheel` 是滾輪輸入事件，不是 Pointer Events
* 它不等於 `scroll`
* 它很常被用來做畫布縮放
* 如果要阻止瀏覽器的預設捲動行為，通常要搭配 `preventDefault()`
* 如果監聽器要呼叫 `preventDefault()`，通常需要設定 `{ passive: false }`
* 在高品質畫布互動中，`wheel` 不只是改變 `zoom`，還常常要搭配「以游標為中心」的視角修正

---

## `wheel` 不等於 `scroll`

這兩者很容易混淆，但角色完全不同：

### `wheel`

表示使用者輸入了滾輪操作。
它是「輸入來源」。

### `scroll`

表示某個容器或頁面實際發生了捲動。
它是「畫面結果」。

所以在畫布系統中，我們常常會在 `wheel` 事件裡攔截瀏覽器原本的捲動行為，改成執行自己的縮放邏輯。

---

## 為什麼常搭配 `preventDefault()`

瀏覽器預設會把滾輪輸入解讀成：

* 頁面上下捲動
* 容器內部捲動
* 觸控板滾動造成的瀏覽器視窗移動

但在畫布系統裡，我們往往不希望它真的去捲頁面，而是希望它改成：

* 放大畫布
* 縮小畫布
* 控制視角

所以通常會這樣做：

```ts
event.preventDefault()
```

它的意思是：

> **阻止瀏覽器原本的預設捲動行為，把這次輸入交給我們自己的畫布邏輯處理。**

---

## 為什麼常搭配 `{ passive: false }`

很多瀏覽器會把某些滾動相關事件監聽器視為 `passive`，也就是預設認為你不會呼叫 `preventDefault()`。

但如果你的程式真的要阻止預設捲動，那就必須明確告訴瀏覽器：

```ts
{ passive: false }
```

否則：

* `preventDefault()` 可能不會生效
* 瀏覽器仍然照常捲動畫面
* 畫布縮放和頁面捲動可能同時發生，互動體驗會很差

所以這一組通常會一起出現：

```ts
event.preventDefault()
```

搭配：

```ts
{ passive: false }
```

---

## 常用欄位

| 欄位                                | 說明                                  | 在畫布互動中的用途                  |
| --------------------------------- | ----------------------------------- | -------------------------- |
| `event.deltaY`                    | 垂直方向的滾動量。往上通常是負值，往下通常是正值            | 最常拿來判斷要放大還是縮小              |
| `event.deltaX`                    | 水平方向的滾動量                            | 在支援橫向平移或特殊控制時會用到           |
| `event.deltaMode`                 | `delta` 的單位模式，可能是 pixel、line 或 page | 用來理解 `delta` 的實際意義，處理跨裝置差異 |
| `event.clientX` / `event.clientY` | 滾輪事件發生時，游標在視窗中的座標                   | 可作為縮放錨點，實作以游標為中心的縮放        |

---

## 這些欄位怎麼一起理解

在畫布縮放場景中，最常見的理解方式是：

1. 先看 `deltaY` 判斷縮放方向

   * `deltaY < 0`：通常代表往上滾，常解讀為放大
   * `deltaY > 0`：通常代表往下滾，常解讀為縮小

2. 再看 `clientX / clientY` 找到當下游標位置
   這可以用來決定縮放的中心點

3. 必要時考慮 `deltaMode`
   因為不同裝置、瀏覽器或輸入方式，`deltaY` 的數值規模可能不同，不能永遠假設它是同一種單位

也就是說，在實作上你通常不是只看「滾了多少」，而是要同時考慮：

* 方向
* 位置
* 單位背景

---

## `deltaMode` 為什麼重要

`deltaY` 看起來像是一個單純數字，但它的單位不一定固定。

`event.deltaMode` 可能表示：

* pixel
* line
* page

這代表不同裝置下，相同的一次滾輪操作，`deltaY` 的大小可能差很多。

因此在實務上要注意：

* 有些情況可以只看正負值，不太在意絕對大小
* 有些情況則需要做正規化，避免不同裝置縮放速度差太多

也就是說：

> `deltaY` 不一定能直接被當成固定倍率使用，它有時只是方向參考，有時還需要額外換算。

## 範例

對應 demo：

* [wheel.html](./demos/wheel.html)

```ts
stage.addEventListener('wheel', (event) => {
  event.preventDefault()

  const delta = event.deltaY < 0 ? 1.08 : 0.92
  zoom = Math.min(Math.max(zoom * delta, 0.4), 2.4)
}, { passive: false })
```

---

## 這段程式在做什麼

這段程式可以拆成三步：

### 1. 阻止預設捲動

```ts
event.preventDefault()
```

這代表不要讓瀏覽器照原本方式去捲動畫面，而是改由程式接手處理。

### 2. 根據 `deltaY` 決定縮放方向

```ts
const delta = event.deltaY < 0 ? 1.08 : 0.92
```

這裡把滾輪方向轉成縮放倍率：

* 往上滾：放大 `1.08`
* 往下滾：縮小 `0.92`

也就是把連續的滾輪輸入，轉成一段段 zoom 變化。

### 3. 限制縮放範圍

```ts
zoom = Math.min(Math.max(zoom * delta, 0.4), 2.4)
```

這一步是為了避免：

* 縮得太小，看不到內容
* 放得太大，失去操作性

所以它不是只改 zoom，而是順便把 zoom 限制在合理範圍內。

---

## 這段程式還少了什麼

這段範例已經足夠展示 `wheel` 的核心概念，但如果要做成更完整的畫布縮放，通常還會再補一件很重要的事：

> **讓縮放盡量以游標所在的位置為中心。**

因為如果你只是單純改 `zoom`，但沒有調整視角位置，那畫面通常會看起來像是「從固定原點縮放」，使用者會覺得焦點跑掉。

高品質畫布縮放通常不只是：

* 改 `zoom`

而是還要一起調整：

* `pan`
* 視窗偏移
* 畫布可視區位置

目標是讓「游標底下那個內容點」在縮放前後盡量維持對齊。

---

## 什麼叫「以游標為中心縮放」

它不是單純把 `clientX/clientY` 記下來而已，而是：

1. 先知道游標目前在視窗中的位置
2. 把這個位置換算成畫布中的某個內容點
3. 縮放後重新調整 pan
4. 讓那個內容點在視窗裡仍然盡量停留在游標附近

這樣使用者才會感覺是「我正在放大我指著的地方」，而不是整張畫布隨便往某個固定角落縮放。

這是畫布、地圖、簡報編輯器很常見的操作體驗重點。

---

## 這種寫法最常見的用途

* 攔截瀏覽器預設捲動
* 把滾輪輸入改成畫布縮放
* 控制 zoom 範圍
* 搭配游標位置做視角修正
* 實作簡報、白板、地圖或設計工具中的縮放互動

## 和 `WorkspaceStage.vue` 的對應

在畫布系統中，`WorkspaceStage.vue` 用 `@wheel.prevent` 把滑鼠滾輪解讀成畫布縮放輸入。

這代表它做的不是傳統頁面捲動，而是把使用者的滾輪輸入轉換成：

* 畫布 zoom 變化
* 視角控制
* 更符合編輯器操作習慣的互動模式

這種做法在以下系統非常常見：

* 畫布編輯器
* 地圖應用
* 白板工具
* 簡報編輯器
* 攝影機視角控制介面
